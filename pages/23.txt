						<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>

					<div id="contentSub"><div id="mw-content-subtitle"><span class="mw-redirectedfrom">(Redirected from <a href="/w/index.php?title=Deep_Web_(search_indexing)&amp;redirect=no" class="mw-redirect" title="Deep Web (search indexing)">Deep Web (search indexing)</a>)</span></div></div>
					
					
					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Content of the World Wide Web that is not indexed by search engines</div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">This article is about the part of the World Wide Web not indexed by traditional search engines. For other uses, see <a href="/wiki/Deep_web_(disambiguation)" class="mw-disambig" title="Deep web (disambiguation)">Deep web (disambiguation)</a>.</div>
<link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1320445320" /><div role="note" class="hatnote navigation-not-searchable">Not to be confused with <a href="/wiki/Dark_web" title="Dark web">Dark web</a>.</div>
<p class="mw-empty-elt">
</p><p>The <b>deep web</b>,<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> <b>invisible web</b>,<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> or <b>hidden web</b><sup id="cite&#95;ref-cthw&#95;3-0" class="reference"><a href="#cite_note-cthw-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> are parts of the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> whose contents are not <a href="/wiki/Search_engine_indexing" title="Search engine indexing">indexed</a> by standard <a href="/wiki/Web_search_engine" class="mw-redirect" title="Web search engine">web search-engine programs</a>.<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> This is in contrast to the "<a href="/wiki/Surface_web" title="Surface web">surface web</a>", which is accessible to anyone using the Internet.<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Computer_science" title="Computer science">Computer scientist</a> Michael K.&#160;Bergman is credited with inventing the term in 2001 as a search-indexing term.<sup id="cite&#95;ref-:0&#95;6-0" class="reference"><a href="#cite_note-:0-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Deep web sites can be accessed by a direct <a href="/wiki/URL" title="URL">URL</a> or <a href="/wiki/IP_address" title="IP address">IP address</a>, but may require entering a password or other security information to access actual content.<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> Uses of deep web sites include <a href="/wiki/Web_mail" class="mw-redirect" title="Web mail">web mail</a>, <a href="/wiki/Online_banking" title="Online banking">online banking</a>, <a href="/wiki/Cloud_storage" title="Cloud storage">cloud storage</a>, restricted-access <a href="/wiki/Social-media" class="mw-redirect" title="Social-media">social-media</a> pages and profiles, and <a href="/wiki/Web_forums" class="mw-redirect" title="Web forums">web forums</a> that require registration for viewing content. It also includes <a href="/wiki/Paywall" title="Paywall">paywalled</a> services such as <a href="/wiki/Video_on_demand" title="Video on demand">video on demand</a> and some online magazines and newspapers.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Terminology">Terminology</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=1" title="Edit section: Terminology"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The first conflation of the terms "deep web" and "<a href="/wiki/Dark_web" title="Dark web">dark web</a>" happened during 2009 when deep web search terminology was discussed together with illegal activities occurring on the <a href="/wiki/Freenet" class="mw-redirect" title="Freenet">Freenet</a> and <a href="/wiki/Darknet" title="Darknet">darknet</a>.<sup id="cite&#95;ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> Those criminal activities include the commerce of personal <a href="/wiki/Password" title="Password">passwords</a>, <a href="/wiki/Identity_document_forgery" title="Identity document forgery">false identity documents</a>, <a href="/wiki/Drug" title="Drug">drugs</a>, <a href="/wiki/Murder" title="Murder">murders</a>, <a href="/wiki/Torture" title="Torture">torture</a>, rape <a href="/wiki/Pornography" title="Pornography">pornography</a>, <a href="/wiki/Firearms" class="mw-redirect" title="Firearms">firearms</a>, and <a href="/wiki/Child_pornography" title="Child pornography">child pornography</a>.<sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Since then, after their use in the media's reporting on the black-market website <a href="/wiki/Silk_Road_(marketplace)" title="Silk Road (marketplace)">Silk Road</a>, media outlets have generally used "deep web" <a href="/wiki/Skunked_term" title="Skunked term">synonymously</a> with the <a href="/wiki/Dark_web" title="Dark web">dark web</a> or <a href="/wiki/Darknet" title="Darknet">darknet</a>, a comparison some reject as inaccurate<sup id="cite&#95;ref-confusion&#95;11-0" class="reference"><a href="#cite_note-confusion-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> and consequently has become an ongoing source of confusion.<sup id="cite&#95;ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> <i><a href="/wiki/Wired_(website)" class="mw-redirect" title="Wired (website)">Wired</a></i> reporters <a href="/wiki/Kim_Zetter" title="Kim Zetter">Kim Zetter</a><sup id="cite&#95;ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/Andy_Greenberg" title="Andy Greenberg">Andy Greenberg</a><sup id="cite&#95;ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> recommend the terms be used in distinct fashions. While the deep web is a reference to any site that cannot be accessed by a traditional search engine, the dark web is a portion of the deep web that has been hidden intentionally and is inaccessible by standard browsers and methods.<sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Non-indexed_content">Non-indexed content</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=2" title="Edit section: Non-indexed content"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Bergman, in a paper on the deep web published in <i>The Journal of Electronic Publishing</i>, mentioned that Jill Ellsworth used the term <i>Invisible Web</i> in 1994 to refer to websites that were not registered with any search engine.<sup id="cite&#95;ref-bergman2001&#95;17-0" class="reference"><a href="#cite_note-bergman2001-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> Bergman cited a January 1996 article by Frank Garcia:<sup id="cite&#95;ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p>
<blockquote>
<p>It would be a site that's possibly reasonably designed, but they didn't bother to register it with any of the search engines. So, no one can find them! You're hidden. I call that the invisible Web.
</p>
</blockquote>
<p>Another early use of the term <i>Invisible Web</i> was by Bruce Mount and Matthew B.&#160;Koll of <a href="/wiki/Personal_Library_Software" class="mw-redirect" title="Personal Library Software">Personal Library Software</a>, in a description of the No.&#160;1 Deep Web program found in a December 1996 press release.<sup id="cite&#95;ref-PLS&#95;19-0" class="reference"><a href="#cite_note-PLS-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The first use of the specific term <i>deep web</i>, now generally accepted, occurred in the aforementioned 2001 Bergman study.<sup id="cite&#95;ref-bergman2001&#95;17-1" class="reference"><a href="#cite_note-bergman2001-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Indexing_methods">Indexing methods</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=3" title="Edit section: Indexing methods"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Methods that prevent web pages from being indexed by traditional search engines may be categorized as one or more of the following:
</p>
<ol><li><b>Contextual web</b>: pages with content varying for different access contexts (e.g., ranges of client IP addresses or previous navigation sequence).</li>
<li><b>Dynamic content</b>: <a href="/wiki/Dynamic_Web_page" class="mw-redirect" title="Dynamic Web page">dynamic pages</a>, which are returned in response to a submitted query or accessed only through a form, especially if open-domain input elements (such as text fields) are used; such fields are hard to navigate without <a href="/wiki/Domain_knowledge" title="Domain knowledge">domain knowledge</a>.</li>
<li><b>Limited access content</b>: sites that limit access to their pages in a technical manner (e.g., using the <a href="/wiki/Robots_Exclusion_Standard" class="mw-redirect" title="Robots Exclusion Standard">Robots Exclusion Standard</a> or <a href="/wiki/CAPTCHA" title="CAPTCHA">CAPTCHAs</a>, or no-store directive, which prohibit search engines from browsing them and creating <a href="/wiki/Web_cache" title="Web cache">cached</a> copies).<sup id="cite&#95;ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> Sites may feature an internal search engine for exploring such pages.<sup id="cite&#95;ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup></li>
<li><b>Non-HTML/text content</b>: textual content encoded in multimedia (image or video) files or specific <a href="/wiki/File_formats" class="mw-redirect" title="File formats">file formats</a> not recognised by search engines.</li>
<li><b>Private web</b>: sites that require registration and login (password-protected resources).</li>
<li><b>Scripted content</b>: pages that are accessible only by links produced by <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> as well as content dynamically downloaded from Web servers via <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash</a> or <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> solutions.</li>
<li><b>Software</b>: certain content is hidden intentionally from the regular Internet, accessible only with special software, such as <a href="/wiki/Tor_(anonymity_network)" class="mw-redirect" title="Tor (anonymity network)">Tor</a>, <a href="/wiki/I2P" title="I2P">I2P</a>, or other darknet software. For example, Tor allows users to access websites using the <a href="/wiki/.onion" title=".onion">.onion</a> server address anonymously, hiding their IP address.</li>
<li><b>Unlinked content</b>: pages which are not linked to by other pages, which may prevent <a href="/wiki/Web_crawling" class="mw-redirect" title="Web crawling">web crawling</a> programs from accessing the content. This content is referred to as pages without <a href="/wiki/Backlink" title="Backlink">backlinks</a> (also known as inlinks). Also, search engines do not always detect all backlinks from searched web pages.</li>
<li><b>Web archives</b>: Web archival services such as the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a> enable users to see archived versions of web pages across time, including websites that have become inaccessible and are not indexed by search engines such as Google.<sup id="cite&#95;ref-:0&#95;6-1" class="reference"><a href="#cite_note-:0-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> The Wayback Machine may be termed a program for viewing the deep web, as web archives that are not from the present cannot be indexed, as past versions of websites are impossible to view by a search. All websites are updated at some time, which is why web archives are considered Deep Web content.<sup id="cite&#95;ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup></li></ol>
<div class="mw-heading mw-heading2"><h2 id="Content_types">Content types</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Deep_web&amp;action=edit&amp;section=4" title="Edit section: Content types"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>While it is not always possible to discover directly a specific web server's content so that it may be indexed, a site potentially can be accessed indirectly (due to <a href="/wiki/Vulnerability_(computing)" class="mw-redirect" title="Vulnerability (computing)">computer vulnerabilities</a>).
</p><p>To discover content on the web, search engines use <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> that follow hyperlinks through known protocol virtual <a href="/wiki/Port_(computer_networking)" title="Port (computer networking)">port numbers</a>. This technique is ideal for discovering content on the surface web but is often ineffective at finding deep web content. For example, these crawlers do not attempt to find dynamic pages that are the result of database queries due to the indeterminate number of queries that are possible.<sup id="cite&#95;ref-:0&#95;6-2" class="reference"><a href="#cite_note-:0-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> It has been noted that this can be overcome (partially) by providing links to query results, but this could unintentionally inflate the popularity of a site of the deep web.
</p><p><a href="/wiki/DeepPeep" title="DeepPeep">DeepPeep</a>, <a href="/wiki/Intute" title="Intute">Intute</a>, <a href="/w/index.php?title=Aleph_Open_Search&amp;action=edit&amp;redlink=1" class="new" title="Aleph Open Search (page does not exist)">Aleph Open Search</a>, <a href="/wiki/Deep_Web_Technologies" title="Deep Web Technologies">Deep Web Technologies</a>, <a href="/wiki/Scirus" title="Scirus">Scirus</a>, and <a href="/wiki/Ahmia.fi" class="mw-redirect" title="Ahmia.fi">Ahmia.fi</a> are a few search engines that have accessed the deep web. Intute ran out of funding and is now a temporary static archive as of July 2011.<sup id="cite&#95;ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> Scirus retired near the end of January 2013.<sup id="cite&#95;ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Researchers have been exploring how the deep web can be crawled in an automatic fashion, including content that can be accessed only by special software such as <a href="/wiki/Tor_(anonymity_network)" class="mw-redirect" title="Tor (anonymity network)">Tor</a>. In 2001, Sriram Raghavan and Hector Garcia-Molina (Stanford Computer Science Department, Stanford University)<sup id="cite&#95;ref-raghavan2000&#95;26-0" class="reference"><a href="#cite_note-raghavan2000-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-27" class="reference"><a href="#cite_note-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup> presented an architectural model for a hidden-Web crawler that used important terms provided by users or collected from the query interfaces to query a Web form and crawl the Deep Web content. Alexandros Ntoulas, Petros Zerfos, and Junghoo Cho of <a href="/wiki/University_of_California,_Los_Angeles" title="University of California, Los Angeles">UCLA</a> created a hidden-Web crawler that automatically generated meaningful queries to issue against search forms.<sup id="cite&#95;ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup> Several form query languages (e.g., DEQUEL<sup id="cite&#95;ref-29" class="reference"><a href="#cite_note-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup>) have been proposed that, besides issuing a query, also allow extraction of structured data from result pages. Another effort is DeepPeep, a project of the <a href="/wiki/University_of_Utah" title="University of Utah">University of Utah</a> sponsored by the <a href="/wiki/National_Science_Foundation" title="National Science Foundation">National Science Foundation</a>, which gathered hidden-web sources (web forms) in different domains based on novel focused crawler techniques.<sup id="cite&#95;ref-30" class="reference"><a href="#cite_note-30"><span class="cite-bracket">&#91;</span>30<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-31" class="reference"><a href="#cite_note-31"><span class="cite-bracket">&#91;</span>31<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Commercial search engines have begun exploring alternative methods to crawl the deep web. The <a href="/wiki/Sitemap_Protocol" class="mw-redirect" title="Sitemap Protocol">Sitemap Protocol</a> (first developed and introduced by Google in 2005) and <a href="/wiki/Open_Archives_Initiative_Protocol_for_Metadata_Harvesting" title="Open Archives Initiative Protocol for Metadata Harvesting">OAI-PMH</a> are mechanisms that allow search engines and other interested parties to discover deep web resources on particular web servers. Both mechanisms allow web servers to advertise the URLs that are accessible on them, thereby allowing automatic discovery of resources that are not linked directly to the surface web. Google's deep web surfacing system computes submissions for each HTML form and adds the resulting HTML pages into the Google search engine index. The surfaced results account for a thousand queries per second to deep web content.<sup id="cite&#95;ref-32" class="reference"><a href="#cite_note-32"><span class="cite-bracket">&#91;</span>32<span class="cite-bracket">&#93;</span></a></sup> In this system, the pre-computation of submissions is done using three algorithms:
</p>
<ol><li>selecting input values for text search inputs that accept keywords,</li>
<li>identifying inputs that accept only values of a specific type (e.g., date) and</li>
<li>selecting a small number of input combinations that generate URLs suitable for inclusion into the Web search index.</li></ol>
<p>In 2008, to facilitate users of <a href="/wiki/Tor_(anonymity_network)#Hidden_services" class="mw-redirect" title="Tor (anonymity network)">Tor hidden services</a> in their access and search of a hidden <a href="/wiki/.onion" title=".onion">.onion</a> suffix, <a href="/wiki/Aaron_Swartz" title="Aaron Swartz">Aaron Swartz</a> designed <a href="/wiki/Tor2web" title="Tor2web">Tor2web</a>—a proxy application able to provide access by means of common web browsers.<sup id="cite&#95;ref-RELEASE&#95;33-0" class="reference"><a href="#cite_note-RELEASE-33"><span class="cite-bracket">&#91;</span>33<span class="cite-bracket">&#93;</span></a></sup> Using this application, deep web links appear as a random sequence of letters followed by the .onion <a href="/wiki/Top-level_domain" title="Top-level domain">top-level domain</a>.
</p>
