					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Identifier for file formats</div>
<p>In <a href="/wiki/Information_and_communications_technology" title="Information and communications technology">information and communications technology</a>, a <b>media type</b>,<sup id="cite&#95;ref-iana&#95;1-0" class="reference"><a href="#cite_note-iana-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-rfc9110section8-3-1&#95;2-0" class="reference"><a href="#cite_note-rfc9110section8-3-1-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> <b>content type</b><sup id="cite&#95;ref-rfc9110section8-3-1&#95;2-1" class="reference"><a href="#cite_note-rfc9110section8-3-1-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-xhtmldatatypes&#95;3-0" class="reference"><a href="#cite_note-xhtmldatatypes-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> or <b>MIME type</b><sup id="cite&#95;ref-iana&#95;1-1" class="reference"><a href="#cite_note-iana-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-whatwgterminology&#95;4-0" class="reference"><a href="#cite_note-whatwgterminology-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-xdgmime&#95;5-0" class="reference"><a href="#cite_note-xdgmime-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> is a two-part identifier for <a href="/wiki/File_format" title="File format">file formats</a> and <a href="/wiki/Content_formats" class="mw-redirect" title="Content formats">content formats</a>. Their purpose is comparable to <a href="/wiki/Filename_extension" title="Filename extension">filename extensions</a> and <a href="/wiki/Uniform_type_identifier" class="mw-redirect" title="Uniform type identifier">uniform type identifiers</a>, in that they identify the intended data format. They are mainly used by technologies underpinning the <a href="/wiki/Internet" title="Internet">Internet</a>, and also used on <a href="/wiki/Linux_desktop" class="mw-redirect" title="Linux desktop">Linux desktop</a> systems.
</p><p>The <a href="/wiki/Internet_Assigned_Numbers_Authority" title="Internet Assigned Numbers Authority">Internet Assigned Numbers Authority (IANA)</a> is the official authority for the standardization and publication of these classifications. Media types are also used by other internet protocols such as <a href="/wiki/HTTP" title="HTTP">HTTP</a>,<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> document file formats such as <a href="/wiki/HTML_element" title="HTML element">HTML</a>,<sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> and the <a href="/wiki/Freedesktop.org" title="Freedesktop.org">XDG</a> specifications implemented by Linux <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a>,<sup id="cite&#95;ref-xdgmime&#95;5-1" class="reference"><a href="#cite_note-xdgmime-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> for similar purposes.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Terminology">Terminology</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=1" title="Edit section: Terminology"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Different internet standards or web standards bodies differ on the preferred term for this type of identifier.
</p><p>The <a href="/wiki/IANA" class="mw-redirect" title="IANA">IANA</a> and <a href="/wiki/IETF" class="mw-redirect" title="IETF">IETF</a> use the term "media type", and consider the term "MIME type" to be obsolete,<sup id="cite&#95;ref-iana&#95;1-2" class="reference"><a href="#cite_note-iana-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> since media types have become used in contexts unrelated to email, such as HTTP. By contrast, the <a href="/wiki/WHATWG" title="WHATWG">WHATWG</a> continues to use the term "MIME type" and discourages use of the term "media type" as ambiguous, since it is used with a different meaning in connection with the <a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">CSS</a> <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">@media</code> feature.<sup id="cite&#95;ref-whatwgterminology&#95;4-1" class="reference"><a href="#cite_note-whatwgterminology-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The <a href="/wiki/HTTP" title="HTTP">HTTP</a> response header for providing the media type is <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">Content-Type</code>.<sup id="cite&#95;ref-rfc9110section8-3-1&#95;2-2" class="reference"><a href="#cite_note-rfc9110section8-3-1-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> The <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> has used <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">ContentType</code> as an <a href="/wiki/XML_Schema_(W3C)#Types" title="XML Schema (W3C)">XML data-type</a> name for a media type.<sup id="cite&#95;ref-xhtmldatatypes&#95;3-1" class="reference"><a href="#cite_note-xhtmldatatypes-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Freedesktop.org" title="Freedesktop.org">XDG</a> specifications implemented by Linux <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a> continue to use the term "MIME type".<sup id="cite&#95;ref-xdgmime&#95;5-2" class="reference"><a href="#cite_note-xdgmime-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Structure">Structure</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=2" title="Edit section: Structure"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">Syntax notations in this section are in <a href="/wiki/Extended_Backus%E2%80%93Naur_form" title="Extended Backus–Naur form">Extended Backus–Naur form</a>, following RFC convention.</div>
<p>A media type consists of a <i>type</i> and a <i>subtype</i>, which is further structured into a <i>tree</i>. A media type can optionally define a <i>suffix</i> and <i>parameters</i>:
</p>
<dl><dd><div class="mw-highlight mw-highlight-lang-ebnf mw-content-ltr" dir="ltr"><pre><span></span><span class="k">mime-type </span><span class="o">=</span> <span class="k">type </span><span class="s2">&quot;/&quot;</span> <span class="p">[</span><span class="k">tree </span><span class="s2">&quot;.&quot;</span><span class="p">]</span> <span class="k">subtype </span><span class="p">[</span><span class="s2">&quot;+&quot;</span> <span class="k">suffix</span><span class="p">]</span> <span class="err">*</span><span class="p">[</span><span class="s2">&quot;;&quot;</span> <span class="k">parameter</span><span class="p">];</span>
</pre></div></dd></dl>
<p>As an example, an HTML file might be designated <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">text/html; charset=UTF-8</code>. In this example, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">text</code> is the type, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">html</code> is the subtype, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">charset=UTF-8</code> is an optional parameter indicating the <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a>.
</p><p>Types, subtypes, and parameter names are case-insensitive. Parameter values are usually case-sensitive, but may be interpreted in a case-insensitive fashion depending on the intended use.<sup id="cite&#95;ref-&#82;FC2045&#95;6-1" class="reference"><a href="#cite_note-RFC2045-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Types">Types</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=3" title="Edit section: Types"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The "type" part defines the broad use of the media type. As of November 1996, the registered types were: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">audio</code>,<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">image</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">message</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">multipart</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">text</code> and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">video</code>.<sup id="cite&#95;ref-&#82;FC2045&#95;6-2" class="reference"><a href="#cite_note-RFC2045-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> By July 2024, the registered types included the foregoing, plus <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">font</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">example</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">model</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">haptics</code>.<sup id="cite&#95;ref-iana&#95;1-3" class="reference"><a href="#cite_note-iana-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup>
</p><p>An unofficial top-level type in common use is <a href="/wiki/Chemical_file_format#The_Chemical_MIME_Project" title="Chemical file format"><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">chemical</code></a>, used for <a href="/wiki/Chemical_file_format" title="Chemical file format">chemical file formats</a>.<sup id="cite&#95;ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> In the context of Linux <a href="/wiki/Desktop_environment" title="Desktop environment">desktop environments</a>, the unofficial top-level types <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">inode</code> (<a href="/wiki/Inode" title="Inode">inodes</a> other than normal files, such as <a href="/wiki/Directory_(computing)" title="Directory (computing)">filesystem directories</a>, <a href="/wiki/Device_file" title="Device file">device files</a> or <a href="/wiki/Symbolic_link" title="Symbolic link">symbolic links</a>),<sup id="cite&#95;ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-content</code> (<a href="/wiki/Removable_media" title="Removable media">removable media</a>, such as <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-content/image-dcf</code> for <a href="/wiki/Design_rule_for_Camera_File_system" title="Design rule for Camera File system">DCF digital cameras</a>),<sup id="cite&#95;ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">package</code> (<a href="/wiki/Package_manager" title="Package manager">package manager</a> packages)<sup id="cite&#95;ref-iconnaming&#95;14-0" class="reference"><a href="#cite_note-iconnaming-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-office</code> (generic categories of <a href="/wiki/Office_productivity_software" class="mw-redirect" title="Office productivity software">office productivity software</a> document)<sup id="cite&#95;ref-iconnaming&#95;14-1" class="reference"><a href="#cite_note-iconnaming-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup> are used.
</p>
<div class="mw-heading mw-heading3"><h3 id="Subtypes">Subtypes</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=4" title="Edit section: Subtypes"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A subtype typically consists of a media format, but it may or must also contain other content, such as a tree prefix, producer, product or suffix, according to the different rules in registration trees.
</p><p>All media types should be registered using the IANA registration procedures. For the efficiency and flexibility of the media type registration process, different structures of subtypes can be registered in registration trees that are distinguished by the use of tree prefixes. Currently, the following trees are created: standard (no prefix), vendor (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">vnd.</code> prefix), personal or vanity (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">prs.</code> prefix), unregistered (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x.</code> prefix). These registration trees were first defined in November 1996 (obsoleted RFC 2048 - currently RFC 6838). New registration trees may be created by <a href="/wiki/Internet_Engineering_Task_Force" title="Internet Engineering Task Force">IETF</a> Standards Action for external registration and management by well-known permanent organizations (e.g. scientific societies).
</p>
<div class="mw-heading mw-heading4"><h4 id="Standards_tree">Standards tree</h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=5" title="Edit section: Standards tree"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The standards tree does not use any tree prefix. Examples are <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">text/javascript</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">image/png</code>.<sup id="cite&#95;ref-&#82;FC6838&#95;16-0" class="reference"><a href="#cite_note-RFC6838-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Registrations in the standards tree must be either associated with IETF specifications approved directly by the IESG, or registered by an IANA recognized standards-related organization.
</p>
<div class="mw-heading mw-heading4"><h4 id="Vendor_tree">Vendor tree</h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=6" title="Edit section: Vendor tree"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The vendor tree includes media types associated with publicly available products. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">vnd.</code> tree prefix. Examples are: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application/vnd.ms-excel</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application/vnd.oasis.opendocument.text</code>.
</p><p>The terms "vendor" and "producer" are considered equivalent in the context. Industry consortia as well as non-commercial entities can register media types in the vendor tree. A registration in the vendor tree may be created by anyone who needs to interchange files associated with some software product or set of products. However, the registration belongs to the vendor or organization producing the software that employs the type being registered, and that vendor or organization can at any time elect to assert ownership of a registration done by a third party.
</p>
<div class="mw-heading mw-heading4"><h4 id="Personal_or_vanity_tree">Personal or vanity tree</h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=7" title="Edit section: Personal or vanity tree"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The personal or vanity tree includes media types associated with non publicly available products or experimental media types. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">prs.</code> tree prefix. Examples are <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">audio/prs.sid</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">image/prs.btif</code>.
</p>
<div class="mw-heading mw-heading4"><h4 id="Unregistered_tree">Unregistered tree</h4><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=8" title="Edit section: Unregistered tree"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The unregistered tree includes media types intended exclusively for use in private environments and only with the active agreement of the parties exchanging them. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x.</code> tree prefix. Examples are <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application/x.foo</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">video/x.bar</code>. Media types in this tree cannot be registered.
</p><p>This type was originally defined in RFC 1590 (published in September 1993) using the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">X-</code> prefix. RFC 2048 (published in November 1996) introduced the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x.</code> prefix, but discouraged use of the unregistered tree, as new personal and vendor trees with relaxed registration requirements are now available. The current RFC 6838 (published in January 2013) maintains the same recommendation, but subtypes prefixed with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">X-</code> are no longer considered to be members of this tree.
</p><p>Media types that have been widely deployed (with a subtype prefixed with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">X-</code>) without being registered, should be, if possible, re-registered with a proper prefixed subtype. If this is not possible, the media type can, after an approval by both the media types reviewer and the IESG, be registered in the standards tree with its unprefixed subtype. <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application/x-www-form-urlencoded</code> is an example of a widely deployed type that ended up registered with the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">x-</code> prefix.<sup id="cite&#95;ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Suffix">Suffix</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=9" title="Edit section: Suffix"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Suffix is an augmentation to the media type definition to additionally specify the underlying structure of that media type, allowing for generic processing based on that structure and independent of the exact type's particular semantics. Media types that make use of a named structured syntax should use the appropriate IANA registered <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">&quot;+&quot;suffix</code> for that structured syntax when they are registered. Unregistered suffixes should not be used (since January 2013). Structured syntax suffix registration procedures are defined in RFC 6838.<sup id="cite&#95;ref-&#82;FC6838&#95;16-1" class="reference"><a href="#cite_note-RFC6838-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The <a href="/wiki/%2Bxml" class="mw-redirect" title="+xml">+xml</a> suffix has been defined since January 2001 (RFC 3023<sup id="cite&#95;ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>), and was formally included in the initial contents of the Structured Syntax Suffix Registry along with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+json</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+ber</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+der</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+fastinfoset</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+wbxml</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+zip</code> in January 2013 (RFC 6839). Subsequent additions include <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+gzip</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+cbor</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+json-seq</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">+cbor-seq</code>.<sup id="cite&#95;ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p>

<div class="mw-heading mw-heading2"><h2 id="Mailcap">Mailcap</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=11" title="Edit section: Mailcap"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1320445320" /><div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/File_association" title="File association">File association</a></div>
<p>Mailcap (derived from the phrase "mail capability") is a type of meta file used to configure how MIME-aware applications such as mail clients and web browsers render files of different MIME-types. The mailcap format is defined by RFC 1524 "A <a href="/wiki/User_agent" title="User agent">User Agent</a> Configuration Mechanism for Multimedia Mail Format Information" but is not defined as an <a href="/wiki/Internet_Standard" title="Internet Standard">Internet standard</a>. It is supported by most Unix systems.
</p><p>Lines can be comments starting with the # character, or a mime-type followed by how to handle that mime type.
</p>
<div class="mw-heading mw-heading2"><h2 id="mime.types">mime.types</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=12" title="Edit section: mime.types"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>An associated file is the <b>mime.types</b> file, which associates filename extensions with a <i>MIME type</i>. If the MIME type is properly set, this is unnecessary, but MIME types may be incorrectly set, or set to a generic type such as <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">application/octet-stream</code>, and <b>mime.types</b> allows one to fall back on the extension in these cases. Similarly, since many file systems do not store MIME type information, but instead rely on the filename extension, a mime.types file is frequently used by web servers to determine MIME type.
</p><p>When <i>viewing</i> a file, these two work together as follows: <b><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">mime.types</code></b> associates an extension with a MIME type, while <code><a href="/wiki/Mailcap" class="mw-redirect" title="Mailcap">mailcap</a></code> associates a MIME type with a program.
</p><p>In UNIX-type systems, the mime.types file is usually located at <code>/etc/<b>mime.types</b></code> and/or <code>$HOME/<b>.mime.types</b></code> and the format is simply that each line is a space-delimited list of a MIME type, followed by zero or more extensions. For example, the HTML type can be associated with the extensions <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">.htm</code> and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">.html</code> by the following line:
</p>
<pre>text/html     htm   html
</pre>
