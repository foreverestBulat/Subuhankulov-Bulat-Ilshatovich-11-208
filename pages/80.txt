					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Maximum rate of data transfer</div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">This article is about use in computing and networking expressed in bits per second. For the concept in signal theory and processing measured in hertz, see <a href="/wiki/Bandwidth_(signal_processing)" title="Bandwidth (signal processing)">Bandwidth (signal processing)</a>. For other uses, see <a href="/wiki/Bandwidth_(disambiguation)" class="mw-redirect mw-disambig" title="Bandwidth (disambiguation)">Bandwidth (disambiguation)</a>.</div>
<p class="mw-empty-elt">
</p><p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b>bandwidth</b> is the maximum rate of data transfer across a given path. Bandwidth may be characterized as <b>network bandwidth</b>,<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> <b>data bandwidth</b>,<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> or <b>digital bandwidth</b>.<sup id="cite&#95;ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>This definition of <i>bandwidth</i> contrasts with usage in <a href="/wiki/Signal_processing" title="Signal processing">signal processing</a>, <a href="/wiki/Wireless_communications" class="mw-redirect" title="Wireless communications">wireless communications</a>, <a href="/wiki/Modem" title="Modem">modem data transmission</a>, <a href="/wiki/Digital_communications" class="mw-redirect" title="Digital communications">digital communications</a>, and <a href="/wiki/Electronics" title="Electronics">electronics</a>, in which <i>bandwidth</i> is used to refer to the <a href="/wiki/Signal_bandwidth" class="mw-redirect" title="Signal bandwidth">signal bandwidth</a> measured in <a href="/wiki/Hertz" title="Hertz">hertz</a>, meaning the frequency range between lowest and highest attainable frequency while meeting a well-defined impairment level in signal power. The actual <a href="/wiki/Bit_rate" title="Bit rate">bit rate</a> that can be achieved depends not only on the signal bandwidth but also on the <a href="/wiki/Noise" title="Noise">noise</a> on the channel.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Network_capacity">Network capacity</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bandwidth_(computing)&amp;action=edit&amp;section=1" title="Edit section: Network capacity"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The term <i>bandwidth</i> sometimes refers to the <a href="/wiki/Net_bit_rate" class="mw-redirect" title="Net bit rate">net bit rate</a>, <i>peak bit rate</i>, <i>information rate</i>, physical-layer <i>useful bit rate</i>, <a href="/wiki/Channel_capacity" title="Channel capacity">channel capacity</a>, or <a href="/wiki/Maximum_throughput" class="mw-redirect" title="Maximum throughput">maximum throughput</a> of a logical or physical communication path in a digital communication system. For example, <a href="/wiki/Bandwidth_test" class="mw-redirect" title="Bandwidth test">bandwidth tests</a> measure the maximum throughput of a computer network. The maximum rate that can be sustained on a link is limited by the <a href="/wiki/Shannon%E2%80%93Hartley" class="mw-redirect" title="Shannon–Hartley">Shannon–Hartley</a> channel capacity for these communication systems, which is dependent on the <a href="/wiki/Bandwidth_(signal_processing)" title="Bandwidth (signal processing)">bandwidth</a> in hertz and the noise on the channel.
</p>
<div class="mw-heading mw-heading2"><h2 id="Network_consumption">Network consumption</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bandwidth_(computing)&amp;action=edit&amp;section=2" title="Edit section: Network consumption"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The <i>consumed bandwidth</i> in bit/s corresponds to achieved <a href="/wiki/Throughput" class="mw-redirect" title="Throughput">throughput</a> or <a href="/wiki/Goodput" title="Goodput">goodput</a>, i.e., the average rate of successful data transfer through a communication path. The consumed bandwidth can be affected by technologies such as <a href="/wiki/Bandwidth_shaping" class="mw-redirect" title="Bandwidth shaping">bandwidth shaping</a>, <a href="/wiki/Bandwidth_management" title="Bandwidth management">bandwidth management</a>, <a href="/wiki/Bandwidth_throttling" title="Bandwidth throttling">bandwidth throttling</a>, <a href="/wiki/Bandwidth_cap" class="mw-redirect" title="Bandwidth cap">bandwidth cap</a>, <a href="/wiki/Bandwidth_allocation" title="Bandwidth allocation">bandwidth allocation</a> (for example <a href="/wiki/Bandwidth_allocation_protocol" title="Bandwidth allocation protocol">bandwidth allocation protocol</a> and <a href="/wiki/Dynamic_bandwidth_allocation" title="Dynamic bandwidth allocation">dynamic bandwidth allocation</a>), etc. A bit stream's bandwidth is proportional to the average consumed signal bandwidth in hertz (the average spectral bandwidth of the analog signal representing the bit stream) during a studied time interval.
</p><p><i>Channel bandwidth</i> may be confused with useful data throughput (or goodput). For example, a channel with <i>x</i> bit/s may not necessarily transmit data at <i>x</i> rate, since protocols, encryption, and other factors can add appreciable overhead. For instance, much internet traffic uses the <a href="/wiki/Transmission_control_protocol" class="mw-redirect" title="Transmission control protocol">transmission control protocol</a> (TCP), which requires a <a href="/wiki/Three-way_handshake" class="mw-redirect" title="Three-way handshake">three-way handshake</a> for each transaction. Although in many modern implementations the protocol is efficient, it does add significant overhead compared to simpler protocols. Also, data packets may be lost, which further reduces the useful data throughput. In general, for any effective digital communication, a framing protocol is needed; overhead and effective throughput depends on implementation. Useful throughput is less than or equal to the actual channel capacity minus implementation overhead.
</p>
<div class="mw-heading mw-heading2"><h2 id="Maximum_throughput">Maximum throughput</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bandwidth_(computing)&amp;action=edit&amp;section=3" title="Edit section: Maximum throughput"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The <a href="/wiki/Asymptotic_bandwidth" class="mw-redirect" title="Asymptotic bandwidth">asymptotic bandwidth</a> (formally <i>asymptotic throughput</i>) for a network is the measure of maximum throughput for a <a href="/wiki/Greedy_source" title="Greedy source">greedy source</a>, for example when the message size (the number of packets per second from a source) approaches close to the maximum amount.<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Asymptotic bandwidths are usually estimated by sending a number of very large messages through the network, measuring the end-to-end throughput. As with other bandwidths, the asymptotic bandwidth is measured in multiples of bits per seconds. Since bandwidth spikes can skew the measurement, carriers often use the 95th <a href="/wiki/Percentile" title="Percentile">percentile</a> method. This method continuously measures bandwidth usage and then removes the top 5 percent.<sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Multimedia">Multimedia</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bandwidth_(computing)&amp;action=edit&amp;section=4" title="Edit section: Multimedia"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Digital bandwidth may also refer to: <a href="/wiki/Bit_rate#Multimedia" title="Bit rate">multimedia bit rate</a> or <a href="/wiki/Average_bitrate" title="Average bitrate">average bitrate</a> after multimedia <a href="/wiki/Data_compression" title="Data compression">data compression</a> (<a href="/wiki/Source_coding" class="mw-redirect" title="Source coding">source coding</a>), defined as the total amount of data divided by the playback time.
</p><p>Due to the impractically high bandwidth requirements of uncompressed <a href="/wiki/Digital_media" title="Digital media">digital media</a>, the required multimedia bandwidth can be significantly reduced with data compression.<sup id="cite&#95;ref-Lee&#95;7-0" class="reference"><a href="#cite_note-Lee-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> The most widely used data compression technique for media bandwidth reduction is the <a href="/wiki/Discrete_cosine_transform" title="Discrete cosine transform">discrete cosine transform</a> (DCT), which was first proposed by <a href="/wiki/Nasir_Ahmed_(engineer)" title="Nasir Ahmed (engineer)">Nasir Ahmed</a> in the early 1970s.<sup id="cite&#95;ref-Stankovic&#95;8-0" class="reference"><a href="#cite_note-Stankovic-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> DCT compression significantly reduces the amount of memory and bandwidth required for digital signals, capable of achieving a <a href="/wiki/Data_compression_ratio" title="Data compression ratio">data compression ratio</a> of up to 100:1 compared to uncompressed media.<sup id="cite&#95;ref-Lea&#95;9-0" class="reference"><a href="#cite_note-Lea-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p>

<p>In <a href="/wiki/Web_hosting_service" title="Web hosting service">web hosting service</a>, the term <i>bandwidth</i> is often used to describe the amount of data transferred to or from the website or server within a prescribed period of time, for example <i>bandwidth consumption accumulated over a month</i> measured in gigabytes per month.<sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> The more accurate phrase used for this meaning of a maximum amount of data transfer each month or given period is <i>monthly data transfer</i>.
</p><p>A similar situation can occur for end-user <a href="/wiki/Internet_service_provider" title="Internet service provider">Internet service providers</a> as well, especially where network capacity is limited (for example in areas with underdeveloped internet connectivity and on wireless networks).
</p>

<div class="mw-heading mw-heading2"><h2 id="Edholm's_law"><span id="Edholm.27s_law"></span>Edholm's law</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bandwidth_(computing)&amp;action=edit&amp;section=7" title="Edit section: Edholm&#039;s law"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1320445320" /><div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Edholm%27s_law" title="Edholm&#39;s law">Edholm's law</a></div>
<p>Edholm's law, proposed by and named after Phil Edholm in 2004,<sup id="cite&#95;ref-Cherry&#95;11-0" class="reference"><a href="#cite_note-Cherry-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> holds that the bandwidth of <a href="/wiki/Telecommunication_network" class="mw-redirect" title="Telecommunication network">telecommunication networks</a> double every 18 months, which has proven to be true since the 1970s.<sup id="cite&#95;ref-Cherry&#95;11-1" class="reference"><a href="#cite_note-Cherry-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-:1&#95;12-0" class="reference"><a href="#cite_note-:1-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> The trend is evident in the cases of <a href="/wiki/Internet" title="Internet">Internet</a>,<sup id="cite&#95;ref-Cherry&#95;11-2" class="reference"><a href="#cite_note-Cherry-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Cellular_network" title="Cellular network">cellular</a> (mobile), <a href="/wiki/Wireless_LAN" title="Wireless LAN">wireless LAN</a> and <a href="/wiki/Personal_area_network" title="Personal area network">wireless personal area networks</a>.<sup id="cite&#95;ref-:1&#95;12-1" class="reference"><a href="#cite_note-:1-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The <a href="/wiki/MOSFET" title="MOSFET">MOSFET</a> (metal–oxide–semiconductor field-effect transistor) is the most important factor enabling the rapid increase in bandwidth.<sup id="cite&#95;ref-Jindal&#95;13-0" class="reference"><a href="#cite_note-Jindal-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> The MOSFET (MOS transistor) was invented by <a href="/wiki/Mohamed_M._Atalla" title="Mohamed M. Atalla">Mohamed M. Atalla</a> and <a href="/wiki/Dawon_Kahng" title="Dawon Kahng">Dawon Kahng</a> at <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a> in 1959,<sup id="cite&#95;ref-computerhistory&#95;14-0" class="reference"><a href="#cite_note-computerhistory-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-Lojek&#95;15-0" class="reference"><a href="#cite_note-Lojek-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-computerhistory-transistor&#95;16-0" class="reference"><a href="#cite_note-computerhistory-transistor-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup> and went on to become the basic building block of modern <a href="/wiki/Telecommunications" title="Telecommunications">telecommunications</a> technology.<sup id="cite&#95;ref-triumph&#95;17-0" class="reference"><a href="#cite_note-triumph-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-Raymer&#95;18-0" class="reference"><a href="#cite_note-Raymer-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup> Continuous <a href="/wiki/MOSFET_scaling" class="mw-redirect" title="MOSFET scaling">MOSFET scaling</a>, along with various advances in MOS technology, has enabled both <a href="/wiki/Moore%27s_law" title="Moore&#39;s law">Moore's law</a> (<a href="/wiki/Transistor_count" title="Transistor count">transistor counts</a> in <a href="/wiki/Integrated_circuit" title="Integrated circuit">integrated circuit</a> chips doubling every two years) and Edholm's law (communication bandwidth doubling every 18 months).<sup id="cite&#95;ref-Jindal&#95;13-1" class="reference"><a href="#cite_note-Jindal-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p>
