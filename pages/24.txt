<p><b>Heritrix</b> is a <a href="/wiki/Web_crawler" title="Web crawler">web crawler</a> designed for <a href="/wiki/Web_archiving" title="Web archiving">web archiving</a>. It was originally written in collaboration between the <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a>, <a href="/wiki/National_Library_of_Norway" title="National Library of Norway">National Library of Norway</a> and <a href="/wiki/National_Library_of_Iceland" class="mw-redirect" title="National Library of Iceland">National Library of Iceland</a>.<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> Heritrix is available under a <a href="/wiki/Free_software_license" class="mw-redirect" title="Free software license">free software license</a> and written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>. The main interface is accessible using a <a href="/wiki/Web_browser" title="Web browser">web browser</a>, and there is a <a href="/wiki/Command-line_tool" class="mw-redirect" title="Command-line tool">command-line tool</a> that can optionally be used to initiate crawls.
</p><p>Heritrix was developed  jointly by the Internet Archive and the Nordic national libraries on specifications written in early 2003. The first official release was in January 2004, and it has been continually improved by employees of the Internet Archive and other interested parties.
</p><p>For many years Heritrix was not the main crawler used to crawl content for the Internet Archive's web collection.<sup id="cite&#95;ref-Kris&#95;3-0" class="reference"><a href="#cite_note-Kris-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> The largest contributor to the collection, as of 2011, is <a href="/wiki/Alexa_Internet" title="Alexa Internet">Alexa Internet</a>.<sup id="cite&#95;ref-Kris&#95;3-1" class="reference"><a href="#cite_note-Kris-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Alexa crawls the web for its own purposes,<sup id="cite&#95;ref-Kris&#95;3-2" class="reference"><a href="#cite_note-Kris-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> using a crawler named <i>ia_archiver</i>. Alexa then donates the material to the Internet Archive.<sup id="cite&#95;ref-Kris&#95;3-3" class="reference"><a href="#cite_note-Kris-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> The Internet Archive itself did some of its own crawling using Heritrix, but only on a smaller scale.<sup id="cite&#95;ref-Kris&#95;3-4" class="reference"><a href="#cite_note-Kris-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Starting in 2008, the Internet Archive began performance improvements to do its own wide scale crawling, and now does collect most of its content.<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag failed verification of its source citation(s). (October 2017)">failed verification</span></a></i>&#93;</sup>
</p>

<div class="mw-heading mw-heading2"><h2 id="Arc_files">Arc files</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Heritrix&amp;action=edit&amp;section=2" title="Edit section: Arc files"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Older versions of Heritrix by default stored the web resources it crawls in an Arc file. This file format is wholly unrelated to <a href="/wiki/ARC_(file_format)" title="ARC (file format)">ARC (file format)</a>.
This format has been used by the Internet Archive since 1996 to store its web archives. More recently it saves by default in the <a href="/wiki/Web_ARChive" class="mw-redirect" title="Web ARChive">WARC file format</a>, which is similar to ARC but more precisely specified and more flexible. Heritrix can also be configured to store files in a directory format similar to the <a href="/wiki/Wget" title="Wget">Wget</a> crawler that uses the URL to name the directory and filename of each resource.
</p><p>An Arc file stores multiple archived resources in a single file in order to avoid managing a large number of small files. The file consists of a sequence of URL records, each with a header containing metadata about how the resource was requested followed by the <a href="/wiki/HTTP_header" class="mw-redirect" title="HTTP header">HTTP header</a> and the response.
</p><p>Example:
</p>
<div class="mw-highlight mw-highlight-lang-http mw-content-ltr" dir="ltr"><pre><span></span><span class="err">filedesc://IA-2006062.arc 0.0.0.0 20060622190110 text/plain 76</span>
<span class="err">1 1 InternetArchive</span>
<span class="err">URL IP-address Archive-date Content-type Archive-length</span>

<span class="err">http://foo.edu:80/hello.html 127.10.100.2 19961104142103 text/html 187</span>
<span class="kr">HTTP</span><span class="o">/</span><span class="m">1.1</span> <span class="m">200</span> <span class="ne">OK</span>
<span class="na">Date</span><span class="o">:</span> <span class="l">Thu, 22 Jun 2006 19:01:15 GMT</span>
<span class="na">Server</span><span class="o">:</span> <span class="l">Apache</span>
<span class="na">Last-Modified</span><span class="o">:</span> <span class="l">Sat, 10 Jun 2006 22:33:11 GMT</span>
<span class="na">Content-Length</span><span class="o">:</span> <span class="l">30</span>
<span class="na">Content-Type</span><span class="o">:</span> <span class="l">text/html</span>

<span class="p">&lt;</span><span class="nt">html</span><span class="p">&gt;</span>
Hello World!!!
<span class="p">&lt;/</span><span class="nt">html</span><span class="p">&gt;</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="Tools_for_processing_Arc_files">Tools for processing Arc files</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Heritrix&amp;action=edit&amp;section=3" title="Edit section: Tools for processing Arc files"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Heritrix includes a command-line tool called <b>arcreader</b> which can be used to extract the contents of an Arc file. The following command lists all the URLs and metadata stored in the given Arc file (in <a rel="nofollow" class="external text" href="https://archive.org/web/researcher/cdx_legend.php">CDX</a> format):
</p>
<pre>arcreader IA-2006062.arc
</pre>
<p>The following command extracts hello.html from the above example assuming the record starts at offset 140:
</p>
<pre>arcreader -o 140 -f dump IA-2006062.arc
</pre>
<p>Other tools:
</p>
<ul><li><a rel="nofollow" class="external text" href="https://web.archive.org/web/20060111160619/http://wiki.lib.umn.edu/DI2/HowToCrawl">Arc processing tools</a></li>
<li><a rel="nofollow" class="external text" href="https://archive-access.sourceforge.net/projects/wera/">WERA (Web ARchive Access)</a> <a rel="nofollow" class="external text" href="https://web.archive.org/web/20110307115632/http://archive-access.sourceforge.net/projects/wera/">Archived</a> 2011-03-07 at the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a></li></ul>
<div class="mw-heading mw-heading2"><h2 id="Command-line_tools">Command-line tools</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Heritrix&amp;action=edit&amp;section=4" title="Edit section: Command-line tools"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Heritrix comes with several command-line tools:
</p>
<ul><li><b>htmlextractor</b> – displays the links Heritrix would extract for a given URL</li>
<li><b>hoppath.pl</b> – recreates the hop path (path of links) to the specified URL from a completed crawl</li>
<li><b>manifest_bundle.pl</b> – bundles up all resources referenced by a crawl manifest file into an uncompressed or compressed tar ball</li>
<li><b>cmdline-jmxclient</b> – enables command-line control of Heritrix</li>
<li><b>arcreader</b> – extracts contents of ARC files (see above)</li></ul>
<p>Further tools are available as part of the Internet Archive's warctools project.<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup>
</p>
