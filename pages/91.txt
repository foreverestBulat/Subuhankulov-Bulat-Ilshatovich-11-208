<p><b>grep</b> is a <a href="/wiki/Command_line_interface" class="mw-redirect" title="Command line interface">command-line</a> utility for searching <a href="/wiki/Plain_text" title="Plain text">text</a> for lines that match a <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a>. Its name comes from the <a href="/wiki/Ed_(text_editor)" title="Ed (text editor)">ed</a> command <code>g/<i>re</i>/p</code> (<b>g</b>lobal, <b>r</b>egular <b>e</b>xpression, <b>p</b>rint), which has the same effect.<sup id="cite&#95;ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-etymology&#95;4-0" class="reference"><a href="#cite_note-etymology-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> grep was originally developed for the <a href="/wiki/Unix" title="Unix">Unix</a> operating system, and is commonly available on <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> and some other systems such as <a href="/wiki/OS-9" title="OS-9">OS-9</a>.<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> The <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a> <a href="/wiki/Command_(computing)" title="Command (computing)">command</a> that <a href="/wiki/Execution_(computing)" title="Execution (computing)">runs</a> the utility has the same name: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" style="" dir="ltr">grep</code>.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Before it was named, grep was a private utility written by <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a> to search files for certain patterns. <a href="/wiki/Doug_McIlroy" class="mw-redirect" title="Doug McIlroy">Doug McIlroy</a>, unaware of its existence, asked Thompson to write such a program. Responding that he would think about such a utility overnight, Thompson actually corrected bugs and made improvements for about an hour on his own program called <code>s</code> (short for "search"). The next day he presented the program to McIlroy, who said it was exactly what he wanted. Thompson's account may explain the belief that grep was written overnight.<sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Thompson wrote the first version in <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> to help <a href="/wiki/Lee_E._McMahon" title="Lee E. McMahon">Lee E. McMahon</a> analyze the text of <i><a href="/wiki/The_Federalist_Papers" title="The Federalist Papers">The Federalist Papers</a></i> to determine authorship of the individual papers.<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> The <a href="/wiki/Ed_(text_editor)" title="Ed (text editor)">ed text editor</a> (also authored by Thompson) had <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a> support but could not be used to search through such a large amount of text, as it loaded the entire file into memory to enable <a href="/wiki/Random_access" title="Random access">random access</a> editing, so Thompson excerpted that regexp code into a standalone tool which would instead process arbitrarily long files sequentially without buffering too much into memory.<sup id="cite&#95;ref-history102&#95;1-2" class="reference"><a href="#cite_note-history102-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> He chose the name because in ed, the command <code>g/<i>re</i>/p</code>, where the <i><code>re</code></i> is the <b>r</b>egular <b>e</b>xpression to match, would print all lines featuring a specified pattern match.<sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> grep was first included in <a href="/wiki/Version_4_Unix" class="mw-redirect" title="Version 4 Unix">Version 4 Unix</a>. Stating that it is "generally cited as <i>the</i> prototypical software tool", McIlroy credited grep with "irrevocably ingraining" Thompson's <a href="/wiki/Tools_philosophy" class="mw-redirect" title="Tools philosophy">tools philosophy</a> in Unix.<sup id="cite&#95;ref-reader&#95;10-0" class="reference"><a href="#cite_note-reader-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Implementations">Implementations</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=2" title="Edit section: Implementations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A variety of grep implementations are available in many operating systems and software development environments.<sup id="cite&#95;ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> Early variants included <code>egrep</code> and <code>fgrep</code>, introduced in <a href="/wiki/Version_7_Unix" title="Version 7 Unix">Version 7 Unix</a>.<sup id="cite&#95;ref-reader&#95;10-1" class="reference"><a href="#cite_note-reader-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> The <code>egrep</code> variant supports an <a href="/wiki/Regular_expression#POSIX_basic_and_extended" title="Regular expression">extended regular expression</a> syntax added by <a href="/wiki/Alfred_Aho" title="Alfred Aho">Alfred Aho</a> after <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a>'s original regular expression implementation.<sup id="cite&#95;ref-Huma1988&#95;12-0" class="reference"><a href="#cite_note-Huma1988-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> The <code>fgrep</code> variant searches for any of a list of <i>fixed</i> strings using the <a href="/wiki/Aho%E2%80%93Corasick_string_matching_algorithm" class="mw-redirect" title="Aho–Corasick string matching algorithm">Aho–Corasick string matching algorithm</a>.<sup id="cite&#95;ref-Meurant1990&#95;13-0" class="reference"><a href="#cite_note-Meurant1990-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> Binaries of these variants exist in modern systems, usually linking to grep or calling grep as a shell script with the appropriate flag added, e.g. <code class="mw-highlight mw-highlight-lang-bash mw-content-ltr" style="" dir="ltr"><span class="nb">exec</span><span class="w"> </span>grep<span class="w"> </span>-E<span class="w"> </span><span class="s2">&quot;</span><span class="nv">$@</span><span class="s2">&quot;</span></code>. Commands <code>egrep</code> and <code>fgrep</code>, while commonly deployed on POSIX systems, to the point the POSIX specification mentions their widespread existence, are actually not part of POSIX.<sup id="cite&#95;ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Other commands contain the word "grep" to indicate they are search tools, typically ones that rely on regular expression matches. The <code><a href="/wiki/Pgrep" title="Pgrep">pgrep</a></code> utility, for instance, displays the processes whose names match a given regular expression.<sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In the <a href="/wiki/Perl" title="Perl">Perl</a> programming language, <code>grep</code> is a built-in function that finds elements in a list that satisfy a certain property.<sup id="cite&#95;ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup> This <a href="/wiki/Higher-order_function" title="Higher-order function">higher-order function</a> is typically named <code><a href="/wiki/Filter_(higher-order_function)" title="Filter (higher-order function)">filter</a></code> or <code>where</code> in other languages.
</p><p><span class="anchor" id="pcregrep"></span>The <code>pcregrep</code> command is an implementation of grep that uses <a href="/wiki/Regular_expression#Perl_and_PCRE" title="Regular expression">Perl regular expression</a> syntax.<sup id="cite&#95;ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup> Similar functionality can be invoked in the GNU version of grep with the <code>-P</code> flag.<sup id="cite&#95;ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p><p><a href="/wiki/Porting" title="Porting">Ports</a> of grep (within <a href="/wiki/Cygwin" title="Cygwin">Cygwin</a> and <a href="/wiki/GnuWin32" title="GnuWin32">GnuWin32</a>, for example) also run under <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. Some versions of Windows feature the similar <code>qgrep</code> or <code><a href="/wiki/Findstr" title="Findstr">findstr</a></code> command.<sup id="cite&#95;ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A grep command is also part of <a href="/wiki/ASCII_Corporation" title="ASCII Corporation">ASCII</a>'s <i>MSX-DOS2 Tools</i> for <a href="/wiki/MSX-DOS" title="MSX-DOS">MSX-DOS</a> version 2.<sup id="cite&#95;ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The <code>grep</code>, <code>egrep</code>, and <code>fgrep</code> commands have also been ported to the <a href="/wiki/IBM_i" title="IBM i">IBM i</a> operating system.<sup id="cite&#95;ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The software <a href="/wiki/Adobe_InDesign" title="Adobe InDesign">Adobe InDesign</a> has functions GREP (since CS3 version (2007)<sup id="cite&#95;ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup>), in the <i>find/change</i> dialog box<sup id="cite&#95;ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup> "GREP" tab, and introduced with InDesign CS4<sup id="cite&#95;ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> in <i>paragraph styles</i><sup id="cite&#95;ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup> "GREP styles".
</p>
<div class="mw-heading mw-heading3"><h3 id="agrep">agrep</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=3" title="Edit section: agrep"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Agrep" title="Agrep">agrep</a></div>
<p><b><code>agrep</code></b> (approximate grep) is an <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> <a href="/wiki/Approximate_string_matching" title="Approximate string matching">approximate string matching</a> program, developed by <a href="/wiki/Udi_Manber" title="Udi Manber">Udi Manber</a> and Sun Wu between 1988 and 1991,<sup id="cite&#95;ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup> for use with the <a href="/wiki/Unix" title="Unix">Unix</a> operating system. It was later ported to <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/DOS" title="DOS">DOS</a>, and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>.
</p><p><i>a</i>grep matches even when the text only <i>approximately</i> fits the search pattern.<sup id="cite&#95;ref-eGrep.SunX&#95;27-0" class="reference"><a href="#cite_note-eGrep.SunX-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup>
</p><p>This following invocation finds <code>netmasks</code> in file <code>myfile</code>, but also any other word that can be derived from it, given no more than two substitutions.
</p>
<pre>agrep -2 netmasks myfile</pre>
<p>This example generates a list of matches with the closest, that is those with the fewest, substitutions listed first. The command flag <code>-B</code> means "best":
</p>
<pre>agrep -B netmasks myfile</pre>
<div class="mw-heading mw-heading2"><h2 id="Usage_as_a_verb">Usage as a verb</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=4" title="Edit section: Usage as a verb"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>In December 2003, the <i><a href="/wiki/Oxford_English_Dictionary" title="Oxford English Dictionary">Oxford English Dictionary</a> Online</i> added "grep" as both a noun and a verb.<sup id="cite&#95;ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A common verb usage is the phrase "You can't grep dead trees"—meaning one can more easily search through digital media, using tools such as grep, than one could with a hard copy (i.e. one made from "dead trees", which in this context is a <a href="/wiki/Dysphemism" title="Dysphemism">dysphemism</a> for paper).<sup id="cite&#95;ref-29" class="reference"><a href="#cite_note-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup>
</p>
