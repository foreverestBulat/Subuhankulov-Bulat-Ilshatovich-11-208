					<div id="contentSub"><div id="mw-content-subtitle"><span class="mw-redirectedfrom">(Redirected from <a href="/w/index.php?title=Web_pages&amp;redirect=no" class="mw-redirect" title="Web pages">Web pages</a>)</span></div></div>
					
					
					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Content provided by a website</div>
<p class="mw-empty-elt">
</p>
<figure class="mw-default-size" typeof="mw:File/Thumb"><a href="/wiki/File:Dead_Zone_Campaign_(2658313491).png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/f/fd/Dead_Zone_Campaign_%282658313491%29.png/250px-Dead_Zone_Campaign_%282658313491%29.png" decoding="async" width="250" height="238" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/fd/Dead_Zone_Campaign_%282658313491%29.png/500px-Dead_Zone_Campaign_%282658313491%29.png 1.5x" data-file-width="1237" data-file-height="1178" /></a><figcaption>The <a href="/wiki/Home_page" title="Home page">home page</a> of <a href="/wiki/NASA" title="NASA">NASA</a> from 2008</figcaption></figure>
<p>A <b>web page</b>, or <b>webpage</b>, is a <a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a> document that is accessed in a <a href="/wiki/Web_browser" title="Web browser">web browser</a>.<sup id="cite&#95;ref-:0&#95;1-0" class="reference"><a href="#cite_note-:0-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> A <a href="/wiki/Website" title="Website">website</a> typically consists of many web pages <a href="/wiki/Hyperlink" title="Hyperlink">linked</a> together under a common <a href="/wiki/Domain_name" title="Domain name">domain name</a>. The term "web page" is therefore a <a href="/wiki/Metaphor" title="Metaphor">metaphor</a> of paper pages bound together into a book.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Navigation">Navigation</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=1" title="Edit section: Navigation"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Web_navigation" title="Web navigation">Web navigation</a></div>
<figure class="mw-default-size" typeof="mw:File/Thumb"><a href="/wiki/File:Platypus_article_on_Vector_2022.png" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e1/Platypus_article_on_Vector_2022.png/250px-Platypus_article_on_Vector_2022.png" decoding="async" width="250" height="160" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e1/Platypus_article_on_Vector_2022.png/500px-Platypus_article_on_Vector_2022.png 1.5x" data-file-width="1694" data-file-height="1087" /></a><figcaption>Each article on the <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> website is a distinct web page. The <a href="/wiki/URL" title="URL">URL</a> is visible in the browser's <a href="/wiki/Address_bar" title="Address bar">address bar</a> at the top.</figcaption></figure>
<p>Each web page is identified by a distinct <a href="/wiki/URL" title="URL">Uniform Resource Locator</a> (URL). When the user inputs a URL into their <a href="/wiki/Web_browser" title="Web browser">web browser</a>, the browser retrieves the necessary content from a <a href="/wiki/Web_server" title="Web server">web server</a> and then <a href="/wiki/Browser_engine" title="Browser engine">transforms</a> it into an interactive visual representation on the user's screen.<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup>
</p><p>If the user <a href="/wiki/Point_and_click" title="Point and click">clicks</a> or <a href="/wiki/Touchscreen" title="Touchscreen">taps</a> a <a href="/wiki/Hyperlink" title="Hyperlink">link</a>, the browser repeats this process to load the new URL, which could be part of the current website or a different one. The browser has <a href="/wiki/Web_browser#Features" title="Web browser">features</a>, such as the <a href="/wiki/Address_bar" title="Address bar">address bar</a>, that indicate which page is displayed.
</p>
<div class="mw-heading mw-heading2"><h2 id="Elements">Elements</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=2" title="Edit section: Elements"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A web page is a <a href="/wiki/Structured_document" title="Structured document">structured document</a>. The core element is a <a href="/wiki/Text_file" title="Text file">text file</a> written in the <a href="/wiki/HTML" title="HTML">HyperText Markup Language</a> (HTML). This specifies the content of the page,<sup id="cite&#95;ref-elems&#95;3-0" class="reference"><a href="#cite_note-elems-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> including <a href="/wiki/Image" title="Image">images</a> and <a href="/wiki/Video" title="Video">video</a>.
</p><p><a href="/wiki/CSS" title="CSS">Cascading Style Sheets</a> (CSS) specify the <a href="/wiki/Separation_of_content_and_presentation" title="Separation of content and presentation">presentation</a> of the page.<sup id="cite&#95;ref-elems&#95;3-1" class="reference"><a href="#cite_note-elems-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> CSS rules can be in separate text files or embedded within the HTML file.
</p><p>The vast majority<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> of pages have <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> <a href="/wiki/Computer_program" title="Computer program">programs</a>, enabling a <a href="/wiki/JavaScript#Examples_of_scripted_behavior" title="JavaScript">wide range</a> of behavior.<sup id="cite&#95;ref-elems&#95;3-2" class="reference"><a href="#cite_note-elems-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> The newer <a href="/wiki/WebAssembly" title="WebAssembly">WebAssembly</a> language can also be used as a <a href="/wiki/WebAssembly#Limitations" title="WebAssembly">supplement</a>.<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The most sophisticated web pages, known as <a href="/wiki/Web_application" title="Web application">web apps</a>, combine these elements in a <a href="/wiki/Programming_complexity" title="Programming complexity">complex manner</a>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Deployment">Deployment</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=3" title="Edit section: Deployment"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>From the perspective of <a href="/wiki/Server-side" class="mw-redirect" title="Server-side">server-side</a> website deployment, there are two types of web pages: <a href="/wiki/Static_web_page" title="Static web page">static</a> and <a href="/wiki/Dynamic_web_page" title="Dynamic web page">dynamic</a>. Static pages are retrieved from the web server's <a href="/wiki/File_system" title="File system">file system</a> without any modification,<sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> while dynamic pages must be created by the server <a href="https://en.wiktionary.org/wiki/on_the_fly" class="extiw" title="wikt:on the fly">on the fly</a>, typically reading from a <a href="/wiki/Database" title="Database">database</a> to fill out a <a href="/wiki/Web_template_system" title="Web template system">template</a>, before being sent to the user's browser.<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> An example of a dynamic page is a <a href="/wiki/Search_engine_results_page" title="Search engine results page">search engine results page</a>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Security">Security</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=4" title="Edit section: Security"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The <a href="/wiki/HTTP" title="HTTP">Hypertext Transfer Protocol</a> (HTTP) is the way a <a href="/wiki/Web_browser" title="Web browser">web browser</a> retrieves webpage content from a <a href="/wiki/Website" title="Website">website</a>. HTTP has two optional security features: <a href="/wiki/HTTPS" title="HTTPS">encrypted mode</a> (HTTPS) greatly reduces the possibility of <a href="/wiki/Man-in-the-middle_attack" title="Man-in-the-middle attack">eavesdropping</a> by a third party,<sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> while <a href="/wiki/Content_Security_Policy" title="Content Security Policy">Content Security Policy</a> protects against <a href="/wiki/Cross-site_scripting" title="Cross-site scripting">cross-site scripting</a> attacks.<sup id="cite&#95;ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> Both features are supported by mainstream web browsers, and many websites use them.
</p>
