					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Hacker technique</div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">Not to be confused with <a href="/wiki/Google_Hacks" title="Google Hacks">Google Hacks</a>.</div>
<p>

<b>Google hacking</b>, also named <b>Google dorking</b>,<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> is a hacker technique that uses Google Search and other <a href="/wiki/Google" title="Google">Google</a> applications to find security holes in the <a href="/wiki/Computer_configuration" title="Computer configuration">configuration</a> and computer code that <a href="/wiki/Website" title="Website">websites</a> are using.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Basics">Basics</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Google_hacking&amp;action=edit&amp;section=1" title="Edit section: Basics"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Google hacking involves using operators in the Google <a href="/wiki/Web_search_engine" class="mw-redirect" title="Web search engine">search engine</a> to locate specific sections of text on websites that are evidence of vulnerabilities, for example specific versions of vulnerable <a href="/wiki/Web_application" title="Web application">Web applications</a>. A search query with <code>intitle:admbook intitle:Fversion filetype:php</code> would locate PHP web pages with the strings "admbook" and "Fversion" in their titles, indicating that the PHP based guestbook Admbook is used, an application with a known <a href="/wiki/Code_injection" title="Code injection">code injection</a> vulnerability. It is normal for <a href="/wiki/Default_(computer_science)" title="Default (computer science)">default</a> installations of applications to include their running version in every page they serve, for example, "Powered by XOOPS 2.2.3 Final", which can be used to search for websites running vulnerable versions.
</p><p>Devices connected to the Internet can be found. A search string such as <code>inurl:"Mode="</code> will find public web cameras.
</p>
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Google_hacking&amp;action=edit&amp;section=2" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The concept of "Google hacking" dates back to August 2002, when Chris Sullo included the "nikto_google.plugin" in the 1.20 release of the <a href="/wiki/Nikto_(vulnerability_scanner)" title="Nikto (vulnerability scanner)">Nikto</a> vulnerability scanner.<sup id="cite&#95;ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> In December 2002 Johnny Long began to collect Google search queries that uncovered <a href="/wiki/Vulnerability_(computing)" class="mw-redirect" title="Vulnerability (computing)">vulnerable systems</a> and/or <a href="/wiki/Data_breach" title="Data breach">sensitive information disclosures</a> – labelling them googleDorks.<sup id="cite&#95;ref-googleDorks2002&#95;4-0" class="reference"><a href="#cite_note-googleDorks2002-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The list of Google Dorks grew into a large dictionary of queries, which were eventually organized into the original Google Hacking Database (GHDB) in 2004.<sup id="cite&#95;ref-ghdb2004&#95;5-0" class="reference"><a href="#cite_note-ghdb2004-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-ghbook2005&#95;6-0" class="reference"><a href="#cite_note-ghbook2005-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Concepts explored in Google hacking have been extended to other <a href="/wiki/Search_engines" class="mw-redirect" title="Search engines">search engines</a>, such as <a href="/wiki/Bing_(search_engine)" class="mw-redirect" title="Bing (search engine)">Bing</a><sup id="cite&#95;ref-bingHackingBF&#95;7-0" class="reference"><a href="#cite_note-bingHackingBF-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/Shodan_(website)#Automated_Search_Tools" title="Shodan (website)">Shodan</a>.<sup id="cite&#95;ref-shodanHackingDB&#95;8-0" class="reference"><a href="#cite_note-shodanHackingDB-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> Automated attack tools<sup id="cite&#95;ref-searchDiggityBF&#95;9-0" class="reference"><a href="#cite_note-searchDiggityBF-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> use custom search dictionaries to find <a href="/wiki/Vulnerability_(computing)" class="mw-redirect" title="Vulnerability (computing)">vulnerable systems</a> and <a href="/wiki/Data_breach" title="Data breach">sensitive information disclosures</a> in public systems that have been indexed by search engines.<sup id="cite&#95;ref-ghHistoryBF&#95;10-0" class="reference"><a href="#cite_note-ghHistoryBF-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>&#160;
</p>
<div class="mw-heading mw-heading2"><h2 id="Protection">Protection</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Google_hacking&amp;action=edit&amp;section=3" title="Edit section: Protection"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Robots.txt" title="Robots.txt">Robots.txt</a> is a well known file for <a href="/wiki/Search_engine_optimization" title="Search engine optimization">search engine optimization</a> and protection against Google dorking. It involves the use of robots.txt to disallow everything or specific endpoints (hackers can still search robots.txt for endpoints) which prevents Google bots from crawling sensitive endpoints such as admin panels.
</p>
