					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Set of web pages that can undermine web crawlers</div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">For trapping real spiders, see <a href="/wiki/Insect_trap" title="Insect trap">Insect trap</a>.</div>
<p>A <b>spider trap</b> (or <b>crawler trap</b>) is a set of web pages that may intentionally or unintentionally be used to cause a <a href="/wiki/Web_crawler" title="Web crawler">web crawler</a> or <a href="/wiki/Search_bot" class="mw-redirect" title="Search bot">search bot</a> to make an infinite number of requests or cause a poorly constructed crawler to crash. Web crawlers are also called <a href="/wiki/Web_spider" class="mw-redirect" title="Web spider">web spiders</a>, from which the name is derived.  Spider traps may be created to "catch" <a href="/wiki/Spambot" title="Spambot">spambots</a> or other crawlers that waste a website's bandwidth.  They may also be created unintentionally by calendars that use <a href="/wiki/Dynamic_web_page" title="Dynamic web page">dynamic pages</a> with links that continually point to the next day or year.
</p><p>Common techniques used include:
</p>
<ul><li>Creation of indefinitely deep <a href="/wiki/Folder_(computing)" class="mw-redirect" title="Folder (computing)">directory</a> structures such as <code>http://example.com/abc/def/abc/def/abc/def/abc/...</code></li>
<li>Dynamic pages that produce an unbounded number of documents for a web crawler to follow. Examples include calendars<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> and algorithmically generated <a href="/wiki/Language_poetry" class="mw-redirect" title="Language poetry">language poetry</a>.<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup></li>
<li>Documents filled with many characters, crashing the <a href="/wiki/Lexical_analysis" title="Lexical analysis">lexical analyzer</a> <a href="/wiki/Parsing" title="Parsing">parsing</a> the document.</li>
<li>Documents with session-id's based on required cookies.</li></ul>
<p>There exists no universal algorithm capable of detecting all spider traps. While certain categories of traps can be identified through automated methods, novel and previously unrecognized traps continue to emerge rapidly.
</p>
<div class="mw-heading mw-heading2"><h2 id="Politeness">Politeness</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Spider_trap&amp;action=edit&amp;section=1" title="Edit section: Politeness"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A spider trap causes a web crawler to enter something like an <a href="/wiki/Infinite_loop" title="Infinite loop">infinite loop</a>,<sup id="cite&#95;ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> which wastes the spider's resources,<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> lowers its productivity, and, in the case of a poorly written crawler, can crash the program.  Polite spiders alternate requests between different hosts, and do not request documents from the same server more than once every several seconds,<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> meaning that a "polite" web crawler is affected to a much lesser degree than an "impolite" crawler.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (November 2022)">citation needed</span></a></i>&#93;</sup>
</p><p>In addition, sites with spider traps usually have a <a href="/wiki/Robots_exclusion_standard" class="mw-redirect" title="Robots exclusion standard">robots.txt</a> telling bots not to go to the trap, so a legitimate "polite" bot would not fall into the trap, whereas an "impolite" bot which disregards the robots.txt settings would be affected by the trap.<sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>
