<p class="mw-empty-elt">
</p>

<p><b>robots.txt</b> is the <a href="/wiki/Filename" title="Filename">filename</a> used for implementing the <b>Robots Exclusion Protocol</b>, a standard used by <a href="/wiki/Website" title="Website">websites</a> to indicate to visiting <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> and other <a href="/wiki/Internet_bot" title="Internet bot">web robots</a> which portions of the website they are allowed to visit.
</p><p>The standard, developed in 1994, relies on <a href="/wiki/Voluntary_compliance" title="Voluntary compliance">voluntary compliance</a>. Malicious bots can use the file as a directory of which pages to visit, though standards bodies discourage countering this with <a href="/wiki/Security_through_obscurity" title="Security through obscurity">security through obscurity</a>. Some archival sites ignore robots.txt. The standard was used in the 1990s to mitigate <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> overload. In the 2020s, websites began denying bots that collect information for <a href="/wiki/Generative_artificial_intelligence" title="Generative artificial intelligence">generative artificial intelligence</a>.
</p><p>The "robots.txt" file can be used in conjunction with <a href="/wiki/Sitemaps" title="Sitemaps">sitemaps</a>, another robot inclusion standard for websites.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="History">History</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=1" title="Edit section: History"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The standard was proposed by <a href="/wiki/Martijn_Koster" title="Martijn Koster">Martijn Koster</a>,<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> when working for <a href="/wiki/Nexor" title="Nexor">Nexor</a><sup id="cite&#95;ref-:0&#95;3-0" class="reference"><a href="#cite_note-:0-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> in February 1994<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> on the <i>www-talk</i> mailing list, the main communication channel for WWW-related activities at the time. <a href="/wiki/Charles_Stross" title="Charles Stross">Charles Stross</a> claims to have provoked Koster to suggest robots.txt, after he wrote a badly behaved web crawler that inadvertently caused a <a href="/wiki/Denial-of-service_attack" title="Denial-of-service attack">denial-of-service attack</a> on Koster's server.<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The standard, initially RobotsNotWanted.txt, allowed <a href="/wiki/Web_developer" title="Web developer">web developers</a> to specify which bots should not access their website or which pages bots should not access. The internet was small enough in 1994 to maintain a complete list of all bots; <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> overload was a primary concern. By June 1994 it had become a <a href="/wiki/De_facto_standard" title="De facto standard"><i>de facto</i> standard</a>;<sup id="cite&#95;ref-Verge&#95;6-0" class="reference"><a href="#cite_note-Verge-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> most complied, including those operated by search engines such as <a href="/wiki/WebCrawler" title="WebCrawler">WebCrawler</a>, <a href="/wiki/Lycos" title="Lycos">Lycos</a>, and <a href="/wiki/AltaVista" title="AltaVista">AltaVista</a>.<sup id="cite&#95;ref-sear&#95;Robo&#95;7-0" class="reference"><a href="#cite_note-sear_Robo-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup>
</p><p>On July 1, 2019, Google announced the proposal of the Robots Exclusion Protocol as an official standard under <a href="/wiki/Internet_Engineering_Task_Force" title="Internet Engineering Task Force">Internet Engineering Task Force</a>.<sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> A proposed standard<sup id="cite&#95;ref-rfc9309&#95;9-0" class="reference"><a href="#cite_note-rfc9309-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> was published in September 2022 as RFC 9309.
</p>
<div class="mw-heading mw-heading2"><h2 id="Standard">Standard</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=2" title="Edit section: Standard"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A site owner wishing to give instructions to web robots places a text file called <style data-mw-deduplicate="TemplateStyles:r886049734">.mw-parser-output .monospaced{font-family:monospace,monospace}</style><span class="monospaced">robots.txt</span> in the root of the web site hierarchy (e.g. <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">https://www.example.com/robots.txt</span>). This text file contains the instructions in a specific format (see examples below). Robots that choose to follow the instructions try to fetch this file and read the instructions before fetching any other file from the <a href="/wiki/Website" title="Website">website</a>. If this file does not exist, web robots assume that the website owner does not wish to place any limitations on crawling the entire site.
</p><p>A robots.txt file contains instructions for bots indicating which web pages they can and cannot access. Robots.txt files are particularly important for web crawlers from search engines such as Google.
</p><p>A robots.txt file on a website will function as a request that specified robots ignore specified files or directories when crawling a site. This might be, for example, out of a preference for privacy from search engine results, or the belief that the content of the selected directories might be misleading or irrelevant to the categorization of the site as a whole, or out of a desire that an application only operates on certain data. Links to pages listed in robots.txt can still appear in search results if they are linked to from a page that is crawled.<sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A robots.txt file covers one <a href="/wiki/Same_origin_policy" class="mw-redirect" title="Same origin policy">origin</a>. For websites with multiple <a href="/wiki/Subdomain" title="Subdomain">subdomains</a>, each subdomain must have its own robots.txt file. If <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">example.com</span> had a robots.txt file but <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">foo.example.com</span> did not, the rules that would apply for <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">example.com</span> would not apply to <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">foo.example.com</span>. In addition, each <a href="/wiki/List_of_URI_schemes" title="List of URI schemes">URI scheme</a> and <a href="/wiki/Port_(computer_networking)" title="Port (computer networking)">port</a> needs its own robots.txt file; <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">http://example.com/robots.txt</span> does not apply to pages under <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">http://example.com:8080/</span> or <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734" /><span class="monospaced">https://example.com/</span>.
</p>
<div class="mw-heading mw-heading2"><h2 id="Compliance">Compliance</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=3" title="Edit section: Compliance"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The robots.txt protocol is widely complied with by bot operators.<sup id="cite&#95;ref-Verge&#95;6-1" class="reference"><a href="#cite_note-Verge-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The robots.txt played a role in the 1999 legal case of <i><a href="/wiki/EBay_v._Bidder%27s_Edge" title="EBay v. Bidder&#39;s Edge">eBay v. Bidder's Edge</a></i>,<sup id="cite&#95;ref-:1&#95;11-0" class="reference"><a href="#cite_note-:1-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> where eBay attempted to block a bot that did not comply with robots.txt, and in May 2000 a court ordered the company operating the bot to stop crawling eBay's servers using any automatic means, by <a href="/wiki/Injunction" title="Injunction">legal injunction</a> on the basis of <a href="/wiki/Trespass_to_chattels" title="Trespass to chattels">trespassing</a>.<sup id="cite&#95;ref-case&#95;12-0" class="reference"><a href="#cite_note-case-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-:1&#95;11-1" class="reference"><a href="#cite_note-:1-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> Bidder's Edge appealed the ruling, but agreed in March 2001 to drop the appeal, pay an undisclosed amount to eBay, and stop accessing eBay's auction information.<sup id="cite&#95;ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2007 <i>Healthcare Advocates v. Harding</i>, a company was sued for accessing protected web pages archived via <a href="/wiki/Wayback_Machine" title="Wayback Machine">The Wayback Machine</a>, despite robots.txt rules denying those pages from the archive. A <a href="/wiki/Pennsylvania" title="Pennsylvania">Pennsylvania</a> court ruled "in this situation, the robots.txt file qualifies as a technological measure" under the <a href="/wiki/Digital_Millennium_Copyright_Act" title="Digital Millennium Copyright Act">DMCA</a>. Due to a malfunction at Internet Archive, Harding could temporarly access these pages from the archive and thus the court found "the Harding firm did not circumvent the protective measure".<sup id="cite&#95;ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2013 <i><a href="/wiki/Associated_Press_v._Meltwater_U.S._Holdings,_Inc." title="Associated Press v. Meltwater U.S. Holdings, Inc.">Associated Press v. Meltwater U.S. Holdings, Inc.</a></i> the Associated Press sued Meltwater for <a href="/wiki/Copyright_infringement" title="Copyright infringement">copyright infringement</a> and misappropriation over copying of AP news items. Meltwater claimed that they did not require a license and that it was <a href="/wiki/Fair_use" title="Fair use">fair use</a>, because the content was freely available and not protected by robots.txt. The court decided in March 2013 that "Meltwater’s copying is not protected by the fair use doctrine", mentioning among several factors that "failure […] to employ the robots.txt protocol did not give Meltwater […] license to copy and publish AP content".<sup id="cite&#95;ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Search_engines">Search engines</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=4" title="Edit section: Search engines"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Some major <a href="/wiki/Search_engine" title="Search engine">search engines</a> following this standard include <a href="/wiki/Ask.com" title="Ask.com">Ask</a>,<sup id="cite&#95;ref-ask-webmasters&#95;20-0" class="reference"><a href="#cite_note-ask-webmasters-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/AOL" title="AOL">AOL</a>,<sup id="cite&#95;ref-about-aol-search&#95;21-0" class="reference"><a href="#cite_note-about-aol-search-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Baidu" title="Baidu">Baidu</a>,<sup id="cite&#95;ref-baidu-spider&#95;22-0" class="reference"><a href="#cite_note-baidu-spider-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Microsoft_Bing" title="Microsoft Bing">Bing</a>,<sup id="cite&#95;ref-bing-blog-robots&#95;23-0" class="reference"><a href="#cite_note-bing-blog-robots-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup>  <a href="/wiki/DuckDuckGo" title="DuckDuckGo">DuckDuckGo</a>,<sup id="cite&#95;ref-duckduckgo-bot&#95;24-0" class="reference"><a href="#cite_note-duckduckgo-bot-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Kagi" title="Kagi">Kagi</a>,<sup id="cite&#95;ref-KagiBot&#95;25-0" class="reference"><a href="#cite_note-KagiBot-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Google_Search" title="Google Search">Google</a>,<sup id="cite&#95;ref-google-webmasters-spec&#95;26-0" class="reference"><a href="#cite_note-google-webmasters-spec-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Yahoo_Search" title="Yahoo Search">Yahoo</a>!,<sup id="cite&#95;ref-yahoo-search-is-bing&#95;27-0" class="reference"><a href="#cite_note-yahoo-search-is-bing-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup> and <a href="/wiki/Yandex_Search" title="Yandex Search">Yandex</a>.<sup id="cite&#95;ref-yandex-robots&#95;28-0" class="reference"><a href="#cite_note-yandex-robots-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Archival_sites">Archival sites</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=5" title="Edit section: Archival sites"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Some web archiving projects ignore robots.txt. <a href="/wiki/Archive_Team" title="Archive Team">Archive Team</a> uses the file to discover more links, such as <a href="/wiki/Sitemap" class="mw-redirect" title="Sitemap">sitemaps</a>.<sup id="cite&#95;ref-29" class="reference"><a href="#cite_note-29"><span class="cite-bracket">&#91;</span>29<span class="cite-bracket">&#93;</span></a></sup> Co-founder <a href="/wiki/Jason_Scott" title="Jason Scott">Jason Scott</a> said that "unchecked, and left alone, the robots.txt file ensures no mirroring or reference for items that may have general use and meaning beyond the website's context."<sup id="cite&#95;ref-30" class="reference"><a href="#cite_note-30"><span class="cite-bracket">&#91;</span>30<span class="cite-bracket">&#93;</span></a></sup> In 2017, the <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a> announced that it would stop complying with robots.txt directives.<sup id="cite&#95;ref-31" class="reference"><a href="#cite_note-31"><span class="cite-bracket">&#91;</span>31<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-Verge&#95;6-2" class="reference"><a href="#cite_note-Verge-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> According to <i><a href="/wiki/Digital_Trends" title="Digital Trends">Digital Trends</a></i>, this followed widespread use of robots.txt to remove historical sites from search engine results, and contrasted with the nonprofit's aim to archive "snapshots" of the internet as it previously existed.<sup id="cite&#95;ref-Internet&#95;Archive&#95;32-0" class="reference"><a href="#cite_note-Internet_Archive-32"><span class="cite-bracket">&#91;</span>32<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Artificial_intelligence">Artificial intelligence</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=6" title="Edit section: Artificial intelligence"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Starting in the 2020s, web operators began using robots.txt to deny access to bots collecting training data for <a href="/wiki/Generative_artificial_intelligence" title="Generative artificial intelligence">generative AI</a>. In 2023, Originality.AI found that 306 of the thousand most-visited websites blocked <a href="/wiki/OpenAI" title="OpenAI">OpenAI</a>'s GPTBot in their robots.txt file and 85 blocked <a href="/wiki/Google" title="Google">Google</a>'s Google-Extended. Many robots.txt files named GPTBot as the only bot explicitly disallowed on all pages. Denying access to GPTBot was common among news websites such as the <a href="/wiki/BBC" title="BBC">BBC</a> and <i><a href="/wiki/The_New_York_Times" title="The New York Times">The New York Times</a></i>. In 2023, blog host <a href="/wiki/Medium_(website)" title="Medium (website)">Medium</a> announced it would deny access to all artificial intelligence web crawlers as "AI companies have leached value from writers in order to spam Internet readers".<sup id="cite&#95;ref-Verge&#95;6-3" class="reference"><a href="#cite_note-Verge-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p>GPTBot complies with the robots.txt standard and gives advice to web operators about how to disallow it, but <i><a href="/wiki/The_Verge" title="The Verge">The Verge</a></i><span class="nowrap" style="padding-left:0.1em;">&#39;</span>s David Pierce said this only began after "training the underlying models that made it so powerful". Also, some bots are used both for search engines and artificial intelligence, and it may be impossible to block only one of these options.<sup id="cite&#95;ref-Verge&#95;6-4" class="reference"><a href="#cite_note-Verge-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> <i><a href="/wiki/404_Media" title="404 Media">404 Media</a></i> reported that companies like <a href="/wiki/Anthropic" title="Anthropic">Anthropic</a> and <a href="/wiki/Perplexity.ai" class="mw-redirect" title="Perplexity.ai">Perplexity.ai</a> circumvented robots.txt by renaming or spinning up new scrapers to replace the ones that appeared on popular <a href="/wiki/Blocklist" class="mw-redirect" title="Blocklist">blocklists</a>.<sup id="cite&#95;ref-33" class="reference"><a href="#cite_note-33"><span class="cite-bracket">&#91;</span>33<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In 2025, the nonprofit <a href="/w/index.php?title=RSL_Collective&amp;action=edit&amp;redlink=1" class="new" title="RSL Collective (page does not exist)">RSL Collective</a> announced the launch of the <a href="/wiki/Really_Simple_Licensing" title="Really Simple Licensing">Really Simple Licensing</a> (RSL) open content licensing standard, allowing web publishers to set terms for AI bots in their robots.txt files. Participating companies at launch included Medium, <a href="/wiki/Reddit" title="Reddit">Reddit</a>, and <a href="/wiki/Yahoo" title="Yahoo">Yahoo</a>.<sup id="cite&#95;ref-tc-10sep2025&#95;34-0" class="reference"><a href="#cite_note-tc-10sep2025-34"><span class="cite-bracket">&#91;</span>34<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-verge-10sep2025&#95;35-0" class="reference"><a href="#cite_note-verge-10sep2025-35"><span class="cite-bracket">&#91;</span>35<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-eg-10sep2025&#95;36-0" class="reference"><a href="#cite_note-eg-10sep2025-36"><span class="cite-bracket">&#91;</span>36<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Security">Security</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=7" title="Edit section: Security"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Despite the use of the terms <i>allow</i> and <i>disallow</i>, the protocol is purely advisory and relies on the compliance of the <a href="/wiki/Web_robot" class="mw-redirect" title="Web robot">web robot</a>; it cannot enforce any of what is stated in the file.<sup id="cite&#95;ref-37" class="reference"><a href="#cite_note-37"><span class="cite-bracket">&#91;</span>37<span class="cite-bracket">&#93;</span></a></sup> Malicious web robots are unlikely to honor robots.txt; some may even use the robots.txt as a guide to find disallowed links and go straight to them. While this is sometimes claimed to be a security risk,<sup id="cite&#95;ref-38" class="reference"><a href="#cite_note-38"><span class="cite-bracket">&#91;</span>38<span class="cite-bracket">&#93;</span></a></sup> this sort of <i><a href="/wiki/Security_through_obscurity" title="Security through obscurity">security through obscurity</a></i> is discouraged by standards bodies. The <a href="/wiki/National_Institute_of_Standards_and_Technology" title="National Institute of Standards and Technology">National Institute of Standards and Technology</a> (NIST) in the United States specifically recommends against this practice: "System security should not depend on the secrecy of the implementation or its components."<sup id="cite&#95;ref-39" class="reference"><a href="#cite_note-39"><span class="cite-bracket">&#91;</span>39<span class="cite-bracket">&#93;</span></a></sup> In the context of robots.txt files, security through obscurity is not recommended as a security technique.<sup id="cite&#95;ref-40" class="reference"><a href="#cite_note-40"><span class="cite-bracket">&#91;</span>40<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Alternatives">Alternatives</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=8" title="Edit section: Alternatives"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Many robots also pass a special <a href="/wiki/User-agent" class="mw-redirect" title="User-agent">user-agent</a> to the web server when fetching content.<sup id="cite&#95;ref-41" class="reference"><a href="#cite_note-41"><span class="cite-bracket">&#91;</span>41<span class="cite-bracket">&#93;</span></a></sup> A web administrator could also configure the server to automatically return failure (or <a href="/wiki/Cloaking" title="Cloaking">pass alternative content</a>) when it detects a connection using one of the robots.<sup id="cite&#95;ref-42" class="reference"><a href="#cite_note-42"><span class="cite-bracket">&#91;</span>42<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-43" class="reference"><a href="#cite_note-43"><span class="cite-bracket">&#91;</span>43<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Some sites, such as <a href="/wiki/Google" title="Google">Google</a>, host a <code>humans.txt</code> file that displays information meant for humans to read.<sup id="cite&#95;ref-44" class="reference"><a href="#cite_note-44"><span class="cite-bracket">&#91;</span>44<span class="cite-bracket">&#93;</span></a></sup> Some sites such as <a href="/wiki/GitHub" title="GitHub">GitHub</a> redirect humans.txt to an <i>About</i> page.<sup id="cite&#95;ref-45" class="reference"><a href="#cite_note-45"><span class="cite-bracket">&#91;</span>45<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Previously, Google had a joke file hosted at <code>/killer-robots.txt</code> instructing <a href="/wiki/Terminator_(character)" title="Terminator (character)">the Terminator</a> not to kill the company founders <a href="/wiki/Larry_Page" title="Larry Page">Larry Page</a> and <a href="/wiki/Sergey_Brin" title="Sergey Brin">Sergey Brin</a>.<sup id="cite&#95;ref-46" class="reference"><a href="#cite_note-46"><span class="cite-bracket">&#91;</span>46<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-47" class="reference"><a href="#cite_note-47"><span class="cite-bracket">&#91;</span>47<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Examples">Examples</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=9" title="Edit section: Examples"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>This example tells all robots that they can visit all files because the wildcard <code>*</code> stands for all robots and the <code>Disallow</code> directive has no value, meaning no pages are disallowed. Search engine giant Google open-sourced their robots.txt parser,<sup id="cite&#95;ref-48" class="reference"><a href="#cite_note-48"><span class="cite-bracket">&#91;</span>48<span class="cite-bracket">&#93;</span></a></sup> and recommends testing and validating rules on the robots.txt file using community-built testers such as Tame the Bots<sup id="cite&#95;ref-49" class="reference"><a href="#cite_note-49"><span class="cite-bracket">&#91;</span>49<span class="cite-bracket">&#93;</span></a></sup> and Real Robots Txt.<sup id="cite&#95;ref-50" class="reference"><a href="#cite_note-50"><span class="cite-bracket">&#91;</span>50<span class="cite-bracket">&#93;</span></a></sup>
</p>
<pre>User-agent: *
Disallow: 
</pre>
<p>This example has the same effect, allowing all files rather than prohibiting none.
</p>
<pre>User-agent: *
Allow: /
</pre>
<p>The same result can be accomplished with an empty or missing robots.txt file.
</p><p>This example tells all robots to stay out of a website:
</p>
<pre>User-agent: *
Disallow: /
</pre>
<p>This example tells all robots not to enter three directories:
</p>
<pre>User-agent: *
Disallow: /cgi-bin/
Disallow: /tmp/
Disallow: /junk/
</pre>
<p>This example tells all robots to stay away from one specific file:
</p>
<pre>User-agent: *
Disallow: /directory/file.html
</pre>
<p>All other files in the specified directory will be processed.
</p><p>This example tells one specific robot to stay out of a website:
</p>
<pre>User-agent: BadBot # replace 'BadBot' with the actual user-agent of the bot
Disallow: /
</pre>
<p>This example tells two specific robots not to enter one specific directory:
</p>
<pre>User-agent: BadBot # replace 'BadBot' with the actual user-agent of the bot
User-agent: Googlebot
Disallow: /private/
</pre>
<p>Example demonstrating how comments can be used:
</p>
<pre># Comments appear after the "#" symbol at the start of a line, or after a directive
User-agent: * # match all bots
Disallow: / # keep them out
</pre>
<p>It is also possible to list multiple robots with their own rules. The actual robot string is defined by the crawler. A few robot operators, such as <a href="/wiki/Google" title="Google">Google</a>, support several user-agent strings that allow the operator to deny access to a subset of their services by using specific user-agent strings.<sup id="cite&#95;ref-google-webmasters-spec&#95;26-1" class="reference"><a href="#cite_note-google-webmasters-spec-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Example demonstrating multiple user-agents:
</p>
<pre>User-agent: googlebot        # all Google services
Disallow: /private/          # disallow this directory

User-agent: googlebot-news   # only the news service
Disallow: /                  # disallow everything

User-agent: *                # any robot
Disallow: /something/        # disallow this directory
</pre>
<div class="mw-heading mw-heading3"><h3 id="The_use_of_the_wildcard_*_in_rules"><span id="The_use_of_the_wildcard_.2A_in_rules"></span>The use of the wildcard * in rules</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=10" title="Edit section: The use of the wildcard * in rules"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The directive <code>Disallow: /something/</code> blocks all files and subdirectories starting with <code>/something/</code>.
</p><p>In contrast using a wildcard, (if supported by the crawler), allows for more complex patterns in specifying paths and files to allow or disallow from crawling, for example <code>Disallow: /something/*/other</code> blocks URLs such as:
</p>
<pre>/something/foo/other
/something/bar/other
</pre>
<p>It would not prevent the crawling of <code>/something/foo/else</code>, as that would not match the pattern.
</p><p>The wildcard <code>*</code> allows greater flexibility but may not be recognized by all crawlers, although it is part of the Robots Exclusion Protocol RFC.<sup id="cite&#95;ref-51" class="reference"><a href="#cite_note-51"><span class="cite-bracket">&#91;</span>51<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A wildcard at the end of a rule in effect does nothing, as that is the standard behaviour.
</p>
<div class="mw-heading mw-heading2"><h2 id="Nonstandard_extensions">Nonstandard extensions</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=11" title="Edit section: Nonstandard extensions"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-heading mw-heading3"><h3 id="Crawl-delay_directive">Crawl-delay directive</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=12" title="Edit section: Crawl-delay directive"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The crawl-delay value is supported by some crawlers to throttle their visits to the host. Since this value is not part of the standard, its interpretation is dependent on the crawler reading it. It is used when the multiple burst of visits from bots is slowing down the host. Yandex interprets the value as the number of seconds to wait between subsequent visits.<sup id="cite&#95;ref-yandex-robots&#95;28-1" class="reference"><a href="#cite_note-yandex-robots-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup> Bing defines crawl-delay as the size of a time window (from 1 to 30 seconds) during which BingBot will access a web site only once.<sup id="cite&#95;ref-bing-crawl-delay&#95;52-0" class="reference"><a href="#cite_note-bing-crawl-delay-52"><span class="cite-bracket">&#91;</span>52<span class="cite-bracket">&#93;</span></a></sup> Google ignores this directive,<sup id="cite&#95;ref-53" class="reference"><a href="#cite_note-53"><span class="cite-bracket">&#91;</span>53<span class="cite-bracket">&#93;</span></a></sup> but provides an interface in its <a href="/wiki/Google_Search_Console" title="Google Search Console">search console</a> for webmasters, to control the <a href="/wiki/Googlebot" title="Googlebot">Googlebot</a>'s subsequent visits.<sup id="cite&#95;ref-54" class="reference"><a href="#cite_note-54"><span class="cite-bracket">&#91;</span>54<span class="cite-bracket">&#93;</span></a></sup>
</p>
<pre>User-agent: bingbot
Allow: /
Crawl-delay: 10
</pre>
<div class="mw-heading mw-heading3"><h3 id="Sitemap">Sitemap</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=13" title="Edit section: Sitemap"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Some crawlers support a <code>Sitemap</code> directive, allowing multiple <a href="/wiki/Sitemaps" title="Sitemaps">Sitemaps</a> in the same <samp>robots.txt</samp> in the form <code>Sitemap: <i>full-url</i></code>:<sup id="cite&#95;ref-55" class="reference"><a href="#cite_note-55"><span class="cite-bracket">&#91;</span>55<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-56" class="reference"><a href="#cite_note-56"><span class="cite-bracket">&#91;</span>56<span class="cite-bracket">&#93;</span></a></sup>
</p>
<pre>Sitemap: http://www.example.com/sitemap.xml</pre>
<div class="mw-heading mw-heading3"><h3 id="Universal_&quot;*&quot;_match"><span id="Universal_.22.2A.22_match"></span>Universal "*" match</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=14" title="Edit section: Universal &quot;*&quot; match"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The <i>Robot Exclusion Standard</i> does not mention the "*" character in the <code>Disallow:</code> statement.<sup id="cite&#95;ref-57" class="reference"><a href="#cite_note-57"><span class="cite-bracket">&#91;</span>57<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Content-Signal">Content-Signal</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=15" title="Edit section: Content-Signal"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Cloudflare" title="Cloudflare">Cloudflare</a> introduced <code>Content-Signal</code><sup id="cite&#95;ref-58" class="reference"><a href="#cite_note-58"><span class="cite-bracket">&#91;</span>58<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-59" class="reference"><a href="#cite_note-59"><span class="cite-bracket">&#91;</span>59<span class="cite-bracket">&#93;</span></a></sup> as a directive to suggest acceptable crawler behavior by type, <code>ai-train</code>, <code>ai-input</code>, and <code>search</code> with values of <code>yes</code> or <code>no</code> for each.<sup id="cite&#95;ref-60" class="reference"><a href="#cite_note-60"><span class="cite-bracket">&#91;</span>60<span class="cite-bracket">&#93;</span></a></sup>
</p>
<pre>Content-Signal: ai-train=no, search=yes, ai-input=no</pre>
<div class="mw-heading mw-heading2"><h2 id="Meta_tags_and_headers">Meta tags and headers</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=16" title="Edit section: Meta tags and headers"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>In addition to root-level robots.txt files, robots exclusion directives can be applied at a more granular level through the use of <a href="/wiki/Robots_meta_tag" class="mw-redirect" title="Robots meta tag">Robots meta tags</a> and X-Robots-Tag HTTP headers. The robots meta tag cannot be used for non-HTML files such as images, text files, or PDF documents. On the other hand, the X-Robots-Tag can be added to non-HTML files by using <a href="/wiki/.htaccess" title=".htaccess">.htaccess</a> and <a href="/wiki/Httpd.conf" class="mw-redirect" title="Httpd.conf">httpd.conf</a> files.<sup id="cite&#95;ref-google-meta&#95;61-0" class="reference"><a href="#cite_note-google-meta-61"><span class="cite-bracket">&#91;</span>61<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="A_&quot;noindex&quot;_meta_tag"><span id="A_.22noindex.22_meta_tag"></span>A "noindex" meta tag</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=17" title="Edit section: A &quot;noindex&quot; meta tag"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">name</span><span class="o">=</span><span class="s">&quot;robots&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;noindex&quot;</span> <span class="p">/&gt;</span>
</pre></div>
<div class="mw-heading mw-heading3"><h3 id="A_&quot;noindex&quot;_HTTP_response_header"><span id="A_.22noindex.22_HTTP_response_header"></span>A "noindex" HTTP response header</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=18" title="Edit section: A &quot;noindex&quot; HTTP response header"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span>X-Robots-Tag: noindex
</pre></div>
<p>The X-Robots-Tag is only effective after the page has been requested and the server responds, and the robots meta tag is only effective after the page has loaded, whereas robots.txt is effective before the page is requested. Thus if a page is excluded by a robots.txt file, any robots meta tags or X-Robots-Tag headers are effectively ignored because the robot will not see them in the first place.<sup id="cite&#95;ref-google-meta&#95;61-1" class="reference"><a href="#cite_note-google-meta-61"><span class="cite-bracket">&#91;</span>61<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading3"><h3 id="Maximum_size_of_a_robots.txt_file">Maximum size of a robots.txt file</h3><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Robots.txt&amp;action=edit&amp;section=19" title="Edit section: Maximum size of a robots.txt file"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The Robots Exclusion Protocol requires crawlers to parse at least 500 kibibytes (512000 bytes) of robots.txt files,<sup id="cite&#95;ref-62" class="reference"><a href="#cite_note-62"><span class="cite-bracket">&#91;</span>62<span class="cite-bracket">&#93;</span></a></sup> which Google maintains as a 500 kibibyte file size restriction for robots.txt files.<sup id="cite&#95;ref-63" class="reference"><a href="#cite_note-63"><span class="cite-bracket">&#91;</span>63<span class="cite-bracket">&#93;</span></a></sup>
</p>
