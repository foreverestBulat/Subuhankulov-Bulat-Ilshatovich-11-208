</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Computer software that provides services to software applications</div>
<p><b>Middleware</b> is a type of <a href="/wiki/Computer_software" class="mw-redirect" title="Computer software">computer software</a> program that provides services to software applications beyond those available from the <a href="/wiki/Operating_system" title="Operating system">operating system</a>. It can be described as "software glue".<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-MW-00&#95;2-0" class="reference"><a href="#cite_note-MW-00-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Middleware makes it easier for <a href="/wiki/Software_developer" class="mw-redirect" title="Software developer">software developers</a> to implement communication and input/output, so they can focus on the specific purpose of their application. It gained popularity in the 1980s as a solution to the problem of how to link newer applications to older legacy systems, although the term had been in use since 1968.<sup id="cite&#95;ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup>
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="In_distributed_applications">In distributed applications</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Middleware&amp;action=edit&amp;section=1" title="Edit section: In distributed applications"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<style data-mw-deduplicate="TemplateStyles:r1320445320">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}@media print{body.ns-0 .mw-parser-output .hatnote{display:none!important}}</style><div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Middleware_(distributed_applications)" title="Middleware (distributed applications)">Middleware (distributed applications)</a></div>
<figure class="mw-default-size" typeof="mw:File/Thumb"><a href="/wiki/File:Middleware_Schema.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/2/20/Middleware_Schema.svg/250px-Middleware_Schema.svg.png" decoding="async" width="250" height="210" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/20/Middleware_Schema.svg/500px-Middleware_Schema.svg.png 1.5x" data-file-width="751" data-file-height="631" /></a><figcaption>Software architecture: Middleware</figcaption></figure>
<p>The term is most commonly used for software that enables communication and management of data in <a href="/wiki/Distributed_application" class="mw-redirect" title="Distributed application">distributed applications</a>. An <a href="/wiki/IETF" class="mw-redirect" title="IETF">IETF</a> workshop in 2000 defined middleware as "those services found above the <a href="/wiki/Transport_layer" title="Transport layer">transport</a> (i.e. over TCP/IP) layer set of services but below the application environment" (i.e. below application-level <a href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">APIs</a>).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2024)">citation needed</span></a></i>&#93;</sup> In this more specific sense <i>middleware</i> can be described as the hyphen ("-") in <i><a href="/wiki/Client-server" class="mw-redirect" title="Client-server">client-server</a></i>, or the <i>-to-</i> in <i><a href="/wiki/Peer-to-peer" title="Peer-to-peer">peer-to-peer</a></i>. Middleware includes <a href="/wiki/Web_server" title="Web server">web servers</a>, <a href="/wiki/Application_server" title="Application server">application servers</a>, <a href="/wiki/Content_management_system" title="Content management system">content management systems</a>, and similar tools that support application development and delivery.<sup id="cite&#95;ref-EtzkornIntro17&#95;4-0" class="reference"><a href="#cite_note-EtzkornIntro17-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>ObjectWeb defines middleware as: "The software layer that lies between the operating system and applications on each side of a distributed computing system in a network."<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> Services that can be regarded as middleware include <a href="/wiki/Enterprise_application_integration" title="Enterprise application integration">enterprise application integration</a>, <a href="/wiki/Data_integration" title="Data integration">data integration</a>, <a href="/wiki/Message_oriented_middleware" class="mw-redirect" title="Message oriented middleware">message oriented middleware</a> (MOM), <a href="/wiki/Object_request_broker" title="Object request broker">object request brokers</a> (ORBs), and the <a href="/wiki/Enterprise_service_bus" title="Enterprise service bus">enterprise service bus</a> (ESB).<sup id="cite&#95;ref-LuckhamEvent11&#95;6-0" class="reference"><a href="#cite_note-LuckhamEvent11-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p><p><a href="/wiki/DBMS" class="mw-redirect" title="DBMS">Database</a> access services are often characterised as middleware. Some of them are language specific implementations and support heterogeneous features and other related communication features.<sup id="cite&#95;ref-SimonOpen14&#95;7-0" class="reference"><a href="#cite_note-SimonOpen14-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Examples of database-oriented middleware include <a href="/wiki/ODBC" class="mw-redirect" title="ODBC">ODBC</a>, <a href="/wiki/JDBC" class="mw-redirect" title="JDBC">JDBC</a>, and <a href="/wiki/Transaction_processing" title="Transaction processing">transaction processing</a> monitors.<sup id="cite&#95;ref-ArregocesData03&#95;8-0" class="reference"><a href="#cite_note-ArregocesData03-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Distributed computing system middleware can loosely be divided into two categories—those that provide human-time services (such as web request servicing) and those that perform in machine-time. This latter middleware is somewhat standardized through the <a href="/wiki/SA_Forum" class="mw-redirect" title="SA Forum">Service Availability Forum</a><sup id="cite&#95;ref-SAFService11&#95;9-0" class="reference"><a href="#cite_note-SAFService11-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> and is commonly used in complex, <a href="/wiki/Embedded_systems" class="mw-redirect" title="Embedded systems">embedded systems</a> within the telecom, defence, and <a href="/wiki/Aerospace" title="Aerospace">aerospace</a> industries.<sup id="cite&#95;ref-JokiahoService12&#95;10-0" class="reference"><a href="#cite_note-JokiahoService12-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Usage_of_middleware">Usage of middleware</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Middleware&amp;action=edit&amp;section=2" title="Edit section: Usage of middleware"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Many categories of middleware have been defined, based on the field in which it is used or the application module it serves. A recent bibliography identified the main categories of middleware as follows:<sup id="cite&#95;ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p>
<ul><li>Transactional: Processing of multiple synchronous/asynchronous transactions, serving as a cluster of associated requests from distributed systems such as bank transactions or credit card payments.</li>
<li>Message-oriented: Message queue and message passing architectures that support synchronous/asynchronous communication.</li>
<li>Procedural: Remote and local architectures to connect, pass, and retrieve software responses of asynchronous systems communications such as a call operation.</li>
<li>Object-oriented: Similar to procedural middleware, but incorporates <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> design principles. Analytically, its software component encompasses object references, exceptions, and inheritance of properties via distributed object requests.</li></ul>
<div class="mw-heading mw-heading2"><h2 id="Other_examples">Other examples</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Middleware&amp;action=edit&amp;section=3" title="Edit section: Other examples"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The term <i>middleware</i> is used in other contexts as well. <i>Middleware</i> is sometimes used in a similar sense to a <a href="/wiki/Software_driver" class="mw-redirect" title="Software driver">software driver</a>, an abstraction layer that hides detail about hardware devices or other software from an application.
</p>
<ul><li>The <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> operating system uses the <a href="/wiki/Linux" title="Linux">Linux</a> kernel at its core, and provides an <a href="/wiki/Application_framework" title="Application framework">application framework</a> that developers incorporate into their applications. In addition, Android provides a middleware layer including <a href="/wiki/Software_library" class="mw-redirect" title="Software library">libraries</a> that provide services such as data storage, screen display, <a href="/wiki/Multimedia" title="Multimedia">multimedia</a>, and web browsing. Because the middleware libraries are <a href="/wiki/Compiler" title="Compiler">compiled</a> to <a href="/wiki/Machine_language" class="mw-redirect" title="Machine language">machine language</a>, services execute quickly. Middleware libraries also implement device-specific functions, so applications and the application framework need not concern themselves with variations between Android devices. Android's middleware layer also contains the <a href="/wiki/Android_Runtime" title="Android Runtime">ART</a> <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> and its core <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> application libraries.<sup id="cite&#95;ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup></li>
<li>Middleware also refers to the software that separates two or more APIs and provides services such as rate-limiting, authentication, and logging.</li>
<li><a href="/wiki/Game_engine" title="Game engine">Game engine</a> software such as <a href="/wiki/Gamebryo" title="Gamebryo">Gamebryo</a> and <a href="/wiki/RenderWare" title="RenderWare">RenderWare</a> are sometimes described as middleware because they provide services to simplify game development.<sup id="cite&#95;ref-MooreIntro06&#95;13-0" class="reference"><a href="#cite_note-MooreIntro06-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup></li>
<li>In simulation technology, <i>middleware</i> is generally used in the context of the <a href="/wiki/High_level_architecture_(simulation)" class="mw-redirect" title="High level architecture (simulation)">high level architecture</a> (HLA) that applies to many distributed simulations. It is a layer of software that lies between the <a href="/wiki/Application_code" class="mw-redirect" title="Application code">application code</a> and the <a href="/wiki/Run-Time_Infrastructure_(simulation)" class="mw-redirect" title="Run-Time Infrastructure (simulation)">run-time infrastructure</a>. Middleware generally consists of a library of functions, and enables a number of applications—simulations or federates in HLA terminology—to <a href="/wiki/Paging" class="mw-redirect" title="Paging">page</a> these functions from the common library rather than recreate them for each application.<sup id="cite&#95;ref-DriraCoop03&#95;14-0" class="reference"><a href="#cite_note-DriraCoop03-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup></li>
<li>In x86 computers, <a href="/wiki/UEFI/BIOS" class="mw-redirect" title="UEFI/BIOS">UEFI/BIOS</a> is a middleware between the operating system and the <a href="/wiki/Computer_hardware" title="Computer hardware">hardware</a>.</li>
<li>Wireless networking developers can use middleware to meet the challenges associated with a <a href="/wiki/Wireless_sensor_network" title="Wireless sensor network">wireless sensor network</a> (WSN). Implementing a middleware application allows WSN developers to integrate operating systems and hardware with the wide variety of currently available applications.<sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup></li>
<li>The <a href="/wiki/QNX" title="QNX">QNX</a> operating system offers middleware for providing multimedia services for use in <a href="/wiki/Automobile" class="mw-redirect" title="Automobile">automobiles</a>, <a href="/wiki/Aircraft" title="Aircraft">aircraft</a>, and other environments.<sup id="cite&#95;ref-QNX08&#95;16-0" class="reference"><a href="#cite_note-QNX08-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup></li>
<li><a href="/wiki/Radio-frequency_identification" title="Radio-frequency identification">Radio-frequency identification</a> (RFID) software toolkits provide middleware to filter noisy and redundant raw data.<sup id="cite&#95;ref-GloverRFID06&#95;17-0" class="reference"><a href="#cite_note-GloverRFID06-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup></li></ul>
