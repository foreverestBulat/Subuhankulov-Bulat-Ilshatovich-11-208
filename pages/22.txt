<p><b>Googlebot</b> is the <a href="/wiki/Web_crawler" title="Web crawler">web crawler</a> software used by <a href="/wiki/Google" title="Google">Google</a> that collects documents from the <a href="/wiki/World_Wide_Web" title="World Wide Web">web</a> to build a searchable index for the <a href="/wiki/Google_Search" title="Google Search">Google Search</a> engine. This name is actually used to refer to two different types of web crawlers: a desktop crawler (to simulate desktop users) and a mobile crawler (to simulate a mobile user).<sup id="cite&#95;ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup>
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Behavior">Behavior</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Googlebot&amp;action=edit&amp;section=1" title="Edit section: Behavior"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>A website will probably be crawled by both Googlebot Desktop and Googlebot Mobile. However starting from September 2020, all sites were switched to mobile-first indexing, meaning Google is crawling the web using a smartphone Googlebot.<sup id="cite&#95;ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> The subtype of Googlebot can be identified by looking at the user agent string in the request. However, both crawler types obey the same product token (useent token) in robots.txt, and so a developer cannot selectively target either Googlebot mobile or Googlebot desktop using robots.txt.
</p><p>Google provides various methods that enable website owners to manage the content displayed in Google's search results. If a <a href="/wiki/Webmaster" title="Webmaster">webmaster</a> chooses to restrict the information on their site available to a Googlebot, or another <a href="/wiki/Web_spider" class="mw-redirect" title="Web spider">spider</a>, they can do so with the appropriate directives in a <a href="/wiki/Robots.txt" title="Robots.txt">robots.txt</a> file,<sup id="cite&#95;ref-tools&#95;3-0" class="reference"><a href="#cite_note-tools-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> or by adding the <a href="/wiki/Meta_element" title="Meta element">meta tag</a> <code>&lt;meta name="Googlebot" content="nofollow" /&gt;</code> to the web page.<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup> Googlebot requests to <a href="/wiki/Web_server" title="Web server">web servers</a> are identifiable by a <a href="/wiki/User-agent" class="mw-redirect" title="User-agent">user-agent</a> string containing "Googlebot" and a host address containing "googlebot.com".<sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup> 
</p><p>Currently, Googlebot follows <a href="/wiki/HREF" class="mw-redirect" title="HREF">HREF</a> links and SRC links.<sup id="cite&#95;ref-tools&#95;3-1" class="reference"><a href="#cite_note-tools-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup>  There is increasing evidence Googlebot can execute JavaScript and parse content generated by <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> calls as well.<sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> There are many theories regarding how advanced Googlebot's ability is to process JavaScript, with opinions ranging from minimal ability derived from custom interpreters.<sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Currently, Googlebot uses a web rendering service (WRS) that is based on the Chromium rendering engine (version 74 as on 7 May 2019).<sup id="cite&#95;ref-evergreen&#95;8-0" class="reference"><a href="#cite_note-evergreen-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> Googlebot discovers pages by harvesting every link on every page that it can find. Unless prohibited by a <a href="/wiki/Nofollow" title="Nofollow">nofollow</a>-tag, it then follows these links to other web pages. New web pages must be linked to from other known pages on the web in order to be crawled and indexed, or manually submitted by the webmaster.
</p><p>A problem that webmasters with low-bandwidth <a href="/wiki/Web_hosting" class="mw-redirect" title="Web hosting">web hosting</a> plans<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2019)">citation needed</span></a></i>&#93;</sup> have often noted with the Googlebot is that it takes up an enormous amount of bandwidth.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (March 2011)">citation needed</span></a></i>&#93;</sup>  This can cause websites to exceed their bandwidth limit and be taken down temporarily. This is especially troublesome for <a href="/wiki/Web_mirror" class="mw-redirect" title="Web mirror">mirror</a> sites which host many <a href="/wiki/Gigabyte" title="Gigabyte">gigabytes</a> of data. Google provides "<a href="/wiki/Google_Search_Console" title="Google Search Console">Search Console</a>" that allow website owners to throttle the crawl rate.<sup id="cite&#95;ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup>
</p><p>How often Googlebot will crawl a site depends on the crawl budget. Crawl budget is an estimation of how typically a website is updated.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2018)">citation needed</span></a></i>&#93;</sup> Technically, Googlebot's development team (Crawling and Indexing team) uses several defined terms internally to take over what "crawl budget" stands for.<sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> Since May 2019, Googlebot uses the latest <a href="/wiki/Chromium_(web_browser)" title="Chromium (web browser)">Chromium</a> rendering engine, which supports <a href="/wiki/ECMAScript_6" class="mw-redirect" title="ECMAScript 6">ECMAScript 6</a> features. This will make the bot a bit more "evergreen" and ensure that it is not relying on an outdated rendering engine compared to browser capabilities.<sup id="cite&#95;ref-evergreen&#95;8-1" class="reference"><a href="#cite_note-evergreen-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Mediabot">Mediabot</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Googlebot&amp;action=edit&amp;section=2" title="Edit section: Mediabot"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><i>Mediabot</i> is the <a href="/wiki/Web_crawler" title="Web crawler">web crawler</a> that <a href="/wiki/Google" title="Google">Google</a> uses for analyzing the content so <a href="/wiki/AdSense" class="mw-redirect" title="AdSense">Google AdSense</a> can serve <a href="/wiki/Contextual_advertising" title="Contextual advertising">contextually relevant</a> advertising to a web page. Mediabot identifies itself with the <a href="/wiki/User_agent" title="User agent">user agent</a> string "Mediapartners-Google/2.1".
</p><p>Unlike other crawlers, Mediabot does not follow links to discover new crawlable URLs, instead only visiting URLs that have included the AdSense code.<sup id="cite&#95;ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup> Where that content resides behind a login, the crawler can be given a log in so that it is able to crawl protected content.<sup id="cite&#95;ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Inspection_Tool_Crawlers">Inspection Tool Crawlers</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Googlebot&amp;action=edit&amp;section=3" title="Edit section: Inspection Tool Crawlers"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><i>InspectionTool</i> is the crawler used by Search testing tools such as the Rich Result Test and URL inspection in <a href="/wiki/Google_Search_Console" title="Google Search Console">Google Search Console</a>. Apart from the user agent and user agent token, it mimics Googlebot.<sup id="cite&#95;ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A guide to the crawlers was independently published.<sup id="cite&#95;ref-14" class="reference"><a href="#cite_note-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> It details four (4) distinctive crawler agents based on <a href="/wiki/Web_server_directory_index" title="Web server directory index">Web server directory index</a> data - one (1) non-chrome and three (3) chrome crawlers.
</p>
